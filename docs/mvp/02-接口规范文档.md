# MVP 接口规范文档

## 一、API 设计规范

### 1.1 设计原则

- **RESTful 风格**: 资源导向，使用 HTTP 动词表达操作
- **版本控制**: URL 路径包含版本号 (`/api/v1/`)
- **统一响应格式**: 所有接口返回统一 JSON 结构
- **错误处理**: 标准化错误码和错误信息
- **认证授权**: JWT Token 认证，RBAC 权限控制

### 1.2 统一响应格式

**成功响应**:
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功",
  "timestamp": "2026-03-11T17:33:05Z",
  "request_id": "req_abc123"
}
```

**错误响应**:
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "参数验证失败",
    "details": [
      {"field": "email", "message": "邮箱格式不正确"}
    ]
  },
  "timestamp": "2026-03-11T17:33:05Z",
  "request_id": "req_abc123"
}
```

### 1.3 通用 HTTP 状态码

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | OK | 请求成功 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 删除成功，无返回内容 |
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未认证或 Token 过期 |
| 403 | Forbidden | 无权限访问 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突（如重复创建） |
| 422 | Unprocessable Entity | 业务逻辑错误 |
| 429 | Too Many Requests | 请求频率超限 |
| 500 | Internal Server Error | 服务器内部错误 |

### 1.4 自定义错误码

| 错误码 | 含义 | HTTP 状态码 |
|--------|------|-------------|
| `VALIDATION_ERROR` | 参数验证失败 | 400 |
| `AUTH_TOKEN_INVALID` | Token 无效或过期 | 401 |
| `AUTH_TOKEN_EXPIRED` | Token 已过期 | 401 |
| `PERMISSION_DENIED` | 权限不足 | 403 |
| `RESOURCE_NOT_FOUND` | 资源不存在 | 404 |
| `RESOURCE_CONFLICT` | 资源冲突 | 409 |
| `BUSINESS_ERROR` | 业务逻辑错误 | 422 |
| `RATE_LIMIT_EXCEEDED` | 频率超限 | 429 |
| `INTERNAL_ERROR` | 内部错误 | 500 |

## 二、认证与授权接口

### 2.1 用户注册

```http
POST /api/v1/auth/register
Content-Type: application/json

{
  "username": "string (required, 3-20 字符)",
  "email": "string (required, 邮箱格式)",
  "password": "string (required, 8-20 字符，包含大小写字母和数字)",
  "company": "string (optional)",
  "phone": "string (optional, 手机号格式)"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "user_id": "usr_abc123",
    "username": "zacon",
    "email": "zacon@example.com",
    "created_at": "2026-03-11T17:33:05Z"
  },
  "message": "注册成功"
}
```

### 2.2 用户登录

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "string (required)",
  "password": "string (required)"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "dGhpcyBpcyBhIHJlZnJl...",
    "token_type": "Bearer",
    "expires_in": 3600,
    "user": {
      "user_id": "usr_abc123",
      "username": "zacon",
      "email": "zacon@example.com",
      "role": "user"
    }
  },
  "message": "登录成功"
}
```

**注意**: `refresh_token` 通过 httpOnly Cookie 返回

### 2.3 刷新 Token

```http
POST /api/v1/auth/refresh
Content-Type: application/json
Cookie: refresh_token=<token>

{
  "refresh_token": "string (required, 或从 Cookie 读取)"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "bmV3IHJlZnJlc2ggdG9r...",
    "token_type": "Bearer",
    "expires_in": 3600
  },
  "message": "Token 刷新成功"
}
```

**注意**: 使用 Rotating 机制，旧 Refresh Token 失效

### 2.4 用户登出

```http
POST /api/v1/auth/logout
Authorization: Bearer <access_token>
```

**响应**:
```json
{
  "success": true,
  "message": "登出成功"
}
```

### 2.5 获取当前用户信息

```http
GET /api/v1/auth/me
Authorization: Bearer <access_token>
```

**响应**:
```json
{
  "success": true,
  "data": {
    "user_id": "usr_abc123",
    "username": "zacon",
    "email": "zacon@example.com",
    "role": "user",
    "company": "Example Corp",
    "created_at": "2026-03-11T17:33:05Z",
    "last_login": "2026-03-11T17:33:05Z"
  }
}
```

## 三、任务管理接口

### 3.1 创建任务

```http
POST /api/v1/tasks
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "content": "string (required, 任务描述)",
  "priority": "integer (optional, 0-2, 默认 0)",
  "agent_id": "string (optional, 指定 Agent，默认 default)",
  "skills": ["string] (optional, 指定技能列表)",
  "metadata": {
    "key": "value"
  } (optional, 自定义元数据)"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "task_id": "tsk_abc123",
    "content": "帮我写一个爬虫",
    "status": "pending",
    "priority": 0,
    "agent_id": "default",
    "created_at": "2026-03-11T17:33:05Z",
    "estimated_time": 300
  },
  "message": "任务创建成功"
}
```

### 3.2 获取任务列表

```http
GET /api/v1/tasks?page=1&page_size=20&status=pending&agent_id=default
Authorization: Bearer <access_token>
```

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | 否 | 页码，默认 1 |
| page_size | integer | 否 | 每页数量，默认 20，最大 100 |
| status | string | 否 | 任务状态过滤 (pending/running/completed/failed) |
| agent_id | string | 否 | Agent ID 过滤 |
| start_date | string | 否 | 开始日期 (ISO 8601) |
| end_date | string | 否 | 结束日期 (ISO 8601) |

**响应**:
```json
{
  "success": true,
  "data": {
    "tasks": [
      {
        "task_id": "tsk_abc123",
        "content": "帮我写一个爬虫",
        "status": "completed",
        "priority": 0,
        "agent_id": "default",
        "created_at": "2026-03-11T17:33:05Z",
        "completed_at": "2026-03-11T17:35:05Z"
      }
    ],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 100,
      "total_pages": 5
    }
  }
}
```

### 3.3 获取任务详情

```http
GET /api/v1/tasks/{task_id}
Authorization: Bearer <access_token>
```

**响应**:
```json
{
  "success": true,
  "data": {
    "task_id": "tsk_abc123",
    "content": "帮我写一个爬虫",
    "status": "completed",
    "priority": 0,
    "agent_id": "default",
    "created_at": "2026-03-11T17:33:05Z",
    "started_at": "2026-03-11T17:33:10Z",
    "completed_at": "2026-03-11T17:35:05Z",
    "result": {
      "success": true,
      "summary": "已完成爬虫开发",
      "artifacts": [
        {"type": "file", "path": "/tmp/spider.py", "name": "爬虫脚本"}
      ]
    },
    "execution_log": [
      {
        "timestamp": "2026-03-11T17:33:10Z",
        "event": "task_started",
        "details": "开始执行任务"
      },
      {
        "timestamp": "2026-03-11T17:35:05Z",
        "event": "task_completed",
        "details": "任务执行成功"
      }
    ]
  }
}
```

### 3.4 取消任务

```http
POST /api/v1/tasks/{task_id}/cancel
Authorization: Bearer <access_token>
```

**响应**:
```json
{
  "success": true,
  "message": "任务已取消"
}
```

**注意**: 仅支持取消 `pending` 或 `running` 状态的任务

### 3.5 重试任务

```http
POST /api/v1/tasks/{task_id}/retry
Authorization: Bearer <access_token>
```

**响应**:
```json
{
  "success": true,
  "data": {
    "task_id": "tsk_new456",
    "original_task_id": "tsk_abc123",
    "status": "pending"
  },
  "message": "任务重试已创建"
}
```

## 九、限流与配额

### 9.1 限流策略

| 接口类别 | 限流规则 | 说明 |
|----------|----------|------|
| 认证接口 | 10 次/分钟/IP | 防止暴力破解 |
| 任务创建 | 60 次/小时/用户 | 防止滥用 |
| 文件上传 | 100MB/小时/用户 | 控制存储成本 |
| 普通查询 | 300 次/分钟/用户 | 保护数据库 |

### 9.2 配额管理

| 用户等级 | 任务配额 | 存储配额 | API 调用配额 |
|----------|----------|----------|--------------|
| 免费版 | 100 次/天 | 1GB | 1000 次/天 |
| 专业版 | 1000 次/天 | 10GB | 10000 次/天 |
| 企业版 | 不限 | 100GB | 不限 |

## 十、版本兼容性

### 10.1 API 版本策略

- **主版本号变更**: 破坏性变更，不向后兼容

## 十一、错误处理最佳实践

### 11.1 客户端错误处理

```javascript
try {
  const response = await fetch('/api/v1/tasks', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ content: '任务内容' })
  });
  
  const data = await response.json();
  
  if (!data.success) {
    // 处理业务错误
    console.error(data.error.message);
    
    // 处理验证错误
    if (data.error.code === 'VALIDATION_ERROR') {
      data.error.details.forEach(detail => {
        showFieldError(detail.field, detail.message);
      });
    }
    
    // 处理 Token 过期
    if (data.error.code === 'AUTH_TOKEN_EXPIRED') {
      await refreshToken();
      // 重试请求
    }
  }
} catch (error) {
  // 处理网络错误
  console.error('Network error:', error);
}
```

### 11.2 重试策略

```javascript
async function requestWithRetry(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, options);
      
      // 5xx 错误可重试
      if (response.status >= 500 && i < maxRetries - 1) {
        await sleep(1000 * Math.pow(2, i)); // 指数退避
        continue;
      }
      
      return response;
    } catch (error) {
      // 网络错误可重试
      if (i < maxRetries - 1) {
        await sleep(1000 * Math.pow(2, i));
        continue;
      }
      throw error;
    }
  }
}
```
