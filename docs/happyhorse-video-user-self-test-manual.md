# 快乐马工作室用户自测手册

本文给普通用户一套可直接照抄执行的自测流程。目标是验证「快乐马内置生图 / 通义生图 → 快乐马视频」的完整链路：先用自己的生图能力生成关键帧分镜图，再让快乐马工作室把这些图片变成视频。用户只需要按步骤输入，提交后等待任务完成即可。

这里涉及几个概念：

- 生图：可以直接用 `happyhorse-video` 的「生图」页或 `hh_image_create` 工具，也可以用独立的 `tongyi-image` 插件。图片会自动保存并生成可供下游使用的 `asset_ids`。
- 图生视频：用 `happyhorse-video` 的 `hh_i2v` 或 Create 页的 `i2v` 模式，把一张首帧图片变成几秒视频。
- `asset_ids`：插件产物的内部资产编号。它像“取件码”，快乐马拿到后能自动找到上一步生成的图片。
- OSS：阿里云对象存储。DashScope（百炼模型服务）需要能从公网 URL 拉取图片/音频/视频，因此快乐马工作室建议配置 OSS。

## 1. 适合谁使用

这份手册适合：

- 想验证快乐马工作室是否安装成功的用户。
- 想先用通义生图出分镜图，再自动生成视频的用户。
- 不想理解复杂参数，只想照着输入、等待结果的用户。

不适合：

- 只想测单个后端 API 的开发者。开发者请看 `docs/happyhorse-video-test-plan.md` 和插件测试。
- 不准备配置百炼 API Key 的用户。没有 API Key 无法真正调用模型。

## 2. 自测前需要准备什么

### 2.1 必备账号和配置

请先准备：

1. 阿里云百炼 DashScope API Key。
2. 阿里云 OSS bucket。
3. OpenAkita 已启动。
4. `happyhorse-video` 插件已安装并启用。
5. 可选：`tongyi-image` 插件已安装并启用。如果只测试快乐马内置生图，则不需要它。

在插件管理中打开插件：

1. 进入 `happyhorse-video` 设置页，填入 DashScope API Key。
2. 如果你也要对比独立通义生图，进入 `tongyi-image` 设置页，填入同一个 DashScope API Key。
3. 在 `happyhorse-video` 设置页填写 OSS：
   - `oss_endpoint`：例如 `oss-cn-beijing.aliyuncs.com`
   - `oss_bucket`：你的 bucket 名称
   - `oss_access_key_id`
   - `oss_access_key_secret`
   - `oss_path_prefix`：保留默认 `happyhorse-video` 即可
4. 在 `happyhorse-video` 设置页点击 Python 依赖安装，确保 `oss2`、`dashscope`、`edge-tts`、`mutagen` 可用。
5. 如果要测长视频拼接，确认 FFmpeg 可用；如果只测 3 段短视频，可以先不测 FFmpeg。

### 2.2 必备素材

基础流程不需要你自己准备图片。我们会先用快乐马内置「生图」页或 `tongyi-image` 生图。

可选素材：

- 如果你想测数字人，需要准备一张清晰正面人像照。
- 如果你想测视频编辑，需要准备一个公网可访问的视频 URL。
- 如果你想测自己的声音，需要准备一个公网可访问的音频 URL；普通测试直接用文字台词即可。

## 3. 推荐自测路线

建议按这个顺序测试：

1. 先测试快乐马内置生图，生成 3 张关键帧。
2. 再测试快乐马图生视频，把 3 张图逐个变成 3 段短视频。
3. 再测试组织模板，让制片人自动调度「编剧 → 生图 → 快乐马视频」。
4. 最后可选测试数字人和长视频。

如果时间有限，只跑第 1、2 步即可确认核心链路。

## 4. 路线 A：手动流程，先生图再生视频

这条路线最稳定，也最容易定位问题。用户手动先生成图片，再把图片交给快乐马。

### A0. 推荐：用快乐马内置生图

操作：

1. 打开 `快乐马工作室`。
2. 进入顶部 `生图` 页签。
3. 选择 `文生图片`。
4. 图片模型选择 `万相 2.7 Pro`，尺寸选择 `2K`。
5. 在提示词里粘贴下面 A2 / A3 / A4 的分镜提示词，每次生成 1 张。
6. 生成完成后到 `任务` 页查看图片。每张图片任务都会产生 `asset_ids`，后续可直接作为图生视频素材。

如果你想对比独立生图插件，也可以继续按 A1 打开 `通义生图`。

### A1. 打开通义生图

操作：

1. 打开 OpenAkita。
2. 进入插件页面。
3. 打开 `通义生图`。
4. 选择文生图模式。
5. 尺寸建议选择 `1024*1792` 或任何 9:16 竖屏尺寸。如果界面没有该尺寸，就选择最接近的竖屏尺寸。

### A2. 生成第一张分镜图

在通义生图提示词输入框粘贴：

```text

```

提交后等待结果。

成功后记录：

- 图片预览是否正常。
- 任务结果中是否有 `asset_id` 或 `asset_ids`。
- 如果界面只显示图片，不显示 `asset_id`，也没关系，组织流程里 runtime 会自动登记。手动流程可复制图片 URL 或本地路径继续测试。

建议把这张图命名为：

```text
镜头一：雨夜街角发光咖啡杯
```

### A3. 生成第二张分镜图

在通义生图提示词输入框粘贴：

```text

```

提交后等待结果。

建议把这张图命名为：

```text
镜头二：咖啡杯霓虹特写
```

### A4. 生成第三张分镜图

在通义生图提示词输入框粘贴：

```text
赛博朋克未来城市咖啡广告，9:16竖屏构图，主角站在高架天桥上喝咖啡，远处是雨夜城市天际线、无人机灯光和霓虹广告牌，主角背影带轮廓光，氛围温暖又未来感，电影感，真实摄影风格
```

提交后等待结果。

建议把这张图命名为：

```text
镜头三：高架天桥城市天际线
```

### A5. 打开快乐马工作室

操作：

1. 打开 `快乐马工作室`。
2. 进入 Create 页。
3. Mode 选择 `图生视频` / `i2v`。
4. Model 保持默认 `happyhorse-1.0-i2v`。
5. Resolution 选择 `720P`。
6. Aspect 选择 `9:16`。
7. Duration 填 `5`。

### A6. 用第一张图生成第一段视频

如果你有第一张图的 URL，把它填入 `first_frame_url`。

提示词粘贴：

```text
镜头一，雨夜街角，主角手持发光咖啡杯缓慢抬起，镜头从中景轻微推进到主角脸部，雨滴落在街面形成霓虹反光，整体赛博朋克电影广告风格，动作自然，画面稳定，保持人物和咖啡杯一致
```

点击生成，等待任务完成。

预期结果：

- Tasks 页出现任务。
- 状态先是 `pending` 或 `running`，最终变为 `succeeded`。
- 结果里有视频预览。
- 结果里有 `video_url` 或本地视频文件路径。

### A7. 用第二张图生成第二段视频

Mode 仍然选择 `i2v`，Model 仍然是 `happyhorse-1.0-i2v`。

把第二张图 URL 填入 `first_frame_url`。

提示词粘贴：

```text
镜头二，咖啡杯特写，镜头缓慢环绕杯身，杯身反射蓝紫色霓虹灯，热气缓慢上升，背景未来城市灯光轻微流动，商业广告质感，细节清晰，运动平滑，保持咖啡杯造型一致
```

点击生成并等待完成。

### A8. 用第三张图生成第三段视频

Mode 仍然选择 `i2v`，Model 仍然是 `happyhorse-1.0-i2v`。

把第三张图 URL 填入 `first_frame_url`。

提示词粘贴：

```text
镜头三，主角站在高架天桥上喝咖啡，镜头从背后缓慢拉远，城市天际线和无人机灯光在雨夜中闪烁，主角轮廓被霓虹边缘光照亮，氛围温暖又未来感，结尾停在品牌广告式构图
```

点击生成并等待完成。

### A9. 手动流程验收标准

通过标准：

1. 通义生图成功生成 3 张分镜图。
2. 快乐马工作室成功生成 3 段视频。
3. 每段视频都能预览或下载。
4. 每段任务都有 `video_url` 或本地路径。
5. 没有出现 `error_kind: auth`、`dependency`、`asset_rejected` 之类错误。

如果失败：

- `auth`：API Key 错误或未保存。
- `dependency`：Python 依赖没装好，去 Settings 安装。
- `asset_rejected`：图片不符合模型要求，换更清晰、无水印、主体明确的图。
- `client`：字段缺失或 URL 不可访问，检查 `first_frame_url` 是否是公网 URL 或 OSS 签名 URL。
- `quota`：百炼账户余额不足或模型调用额度不足。

## 5. 路线 B：组织模板全自动流程

这条路线更接近真实使用方式。用户只输入一次需求，组织会自动调度编剧、通义生图和快乐马视频。

### B1. 创建组织

操作：

1. 打开「组织编排」。
2. 点击「从模板创建」。
3. 选择「百炼 AIGC 视频创作工作室」。
4. 创建后把组织名称改成：

```text
快乐马视频工作室测试
```

5. 点击启动组织。

### B2. 在指挥台输入完整任务

在组织指挥台输入：

```text
请制作一个 15 秒竖屏 AIGC 短片，主题是「未来城市咖啡广告」。

请严格按下面流程执行：
1. 先让编剧写一个 3 镜头脚本，每个镜头 5 秒，包含画面描述、镜头运动、字幕或旁白。
2. 再让通义生图工作台为每个镜头生成 1 张 9:16 竖屏关键帧分镜图。
3. 通义生图成功后，请把每张图返回的 asset_ids 传给快乐马视频工作台。
4. 快乐马视频工作台必须逐镜调用 hh_i2v，每次只消费当前镜头的 from_asset_ids，模型使用 happyhorse-1.0-i2v，duration=5，resolution=720P，aspect_ratio=9:16。
5. 最终交付时请给我：完整脚本、3 张分镜图附件、3 段视频附件，并说明每个附件对应哪个镜头。

整体风格：赛博朋克、霓虹灯、雨夜、未来城市、电影广告质感。
```

提交后等待即可。

### B3. 等待过程应该看到什么

正常情况下会依次出现：

1. 制片人接收任务。
2. 制片人委派编剧写脚本。
3. 编剧交付 3 镜头脚本。
4. 制片人委派通义生图工作台出图。
5. 通义生图工作台调用 `tongyi_image_create`。
6. 通义生图返回图片和 `asset_ids`。
7. 制片人委派快乐马视频工作台。
8. 快乐马视频工作台逐个调用 `hh_i2v`。
9. 快乐马视频返回视频。
10. 制片人汇总交付。

等待期间不用重复催促。视频任务通常需要几十秒到数分钟，取决于百炼排队情况。

### B4. 最终结果应该包含什么

最终交付中应该包含：

1. 3 镜头脚本。
2. 3 张分镜图附件。
3. 3 段视频附件或视频链接。
4. 每个镜头的说明。
5. 快乐马任务的 `task_id` 或 `video_url`。

如果最终只有文字，没有图片或视频附件，说明工作台联动没有跑完整，需要查看任务流中卡在哪一步。

## 6. 路线 C：主聊天窗口直接调用组织

如果组织已经创建并启动，可以在主聊天里直接输入：

```text
@org:快乐马视频工作室测试 请制作一个 15 秒竖屏 AIGC 短片，主题是「未来城市咖啡广告」。

流程要求：
1. 编剧先写 3 镜头脚本。
2. 通义生图工作台为每个镜头生成关键帧图片。
3. 把通义生图返回的 asset_ids 传给快乐马视频工作台。
4. 快乐马视频工作台逐镜调用 hh_i2v，并使用 from_asset_ids。
5. 最终交付脚本、分镜图附件、视频附件。

风格：赛博朋克、霓虹灯、雨夜、电影感，9:16 竖屏。
```

然后等待组织返回结果。

## 7. 路线 D：只测快乐马自己的文生视频

如果你暂时不想用生图，也可以只测试快乐马是否能直接生成视频。

打开快乐马工作室 Create 页：

1. Mode 选择 `t2v`。
2. Model 保持 `happyhorse-1.0-t2v`。
3. Duration 填 `5`。
4. Resolution 选 `720P`。
5. Aspect 选 `16:9`。

提示词粘贴：

```text
清晨海边，太阳从海平面升起，金色阳光洒在浪花上，镜头从浪花特写缓慢拉远到大海全景，电影感，真实摄影，高级广告质感，画面稳定，色彩温暖
```

点击生成，等待完成。

通过标准：

1. 任务状态变成 `succeeded`。
2. 能看到一段 5 秒左右视频。
3. 成本预估接近 5 × 0.90 元。

## 8. 路线 E：数字人照片说话

这条路线需要你准备一张人像。

### E1. 准备素材

准备一张图片：

- 真人正面照。
- 单人。
- 脸部清晰。
- 不要戴大面积遮挡物。
- 不要使用卡通、动物、多人合照。

### E2. 提交照片说话

打开快乐马工作室 Create 页：

1. Mode 选择 `photo_speak`。
2. Model 保持 `wan2.2-s2v`。
3. `image_url` 填你上传后得到的图片 URL。
4. `voice_id` 选择 `longxiaochun_v2` 或任意 CosyVoice 音色。
5. `text` 粘贴：

```text
你好，我是快乐马工作室生成的数字人。现在正在为你演示阿里云百炼的视频能力。
```

点击生成并等待。

通过标准：

1. 先生成语音。
2. 再生成数字人视频。
3. 人物口型与文字内容大致同步。

常见失败：

- `asset_rejected`：人脸不清晰或不是正面人像。
- `dependency`：TTS 依赖未安装。
- `quota`：百炼额度不足。

## 9. 路线 F：长视频分镜拼接

长视频依赖 FFmpeg，建议在前面流程都通过后再测。

打开快乐马工作室 Storyboard 页。

故事输入：

```text
一只橘猫在阳光下的花园里追蝴蝶，先从草地中探头，再穿过花丛，最后跳到木椅上看向镜头。整体氛围温暖治愈，适合做 15 秒短片。
```

参数：

- 总时长：`15`
- 每段时长：`5`
- 基础模型：`happyhorse-1.0-i2v`
- 串接方式：`serial`

点击 AI 拆分镜，确认生成 3 段后点击提交生成。

通过标准：

1. Tasks 页出现 3 个分段任务。
2. 每段任务最终变成 `succeeded`。
3. 如果 FFmpeg 可用，会生成拼接后的视频。

## 10. 用户只需等待的时间参考

通义生图：

- 单张图通常几十秒到 2 分钟。
- 3 张图建议预留 3 到 6 分钟。

快乐马图生视频：

- 每段 5 秒视频通常几十秒到数分钟。
- 3 段视频建议预留 5 到 15 分钟。

组织自动流程：

- 包含编剧、3 张图、3 段视频，建议预留 10 到 25 分钟。

如果超过 30 分钟还没有结果，请检查 Tasks 页状态、错误信息和插件日志。

## 11. 最终验收清单

完成一次完整自测后，请确认：

1. `tongyi-image` 能生成图片。
2. 图片能被登记为 `asset_ids`。
3. `happyhorse-video` 能用 `from_asset_ids` 或 `first_frame_url` 生成视频。
4. 生成的视频能预览或下载。
5. 组织模板能自动完成「编剧 → 生图 → 视频」。
6. 最终交付不是纯文本，而是包含脚本、图片附件、视频附件。

全部满足即可认为快乐马工作室基础功能可用。

## 12. 排错速查

### 12.1 提示没有 API Key

处理：

1. 打开插件 Settings。
2. 填写 DashScope API Key。
3. 点击保存。
4. 点击测试连接。

### 12.2 提示 OSS 未配置或 URL 不可访问

处理：

1. 打开快乐马工作室 Settings。
2. 填写 OSS endpoint、bucket、AccessKey ID、AccessKey Secret。
3. 保存后重新上传图片。
4. 不要直接把本地文件路径交给 DashScope。

### 12.3 图片被拒绝

常见原因：

1. 图片太模糊。
2. 图片含水印。
3. 图片不是人脸但选择了数字人模式。
4. 图片中有多个人脸。
5. 图片内容触发审核。

处理：

1. 换更清晰的图。
2. 简化画面主体。
3. 数字人模式用正面单人照。

### 12.4 视频一直 pending 或 running

可能原因：

1. 百炼队列繁忙。
2. 同一个 API Key 同时只有一个异步视频任务在跑。
3. 网络暂时不稳定。

处理：

1. 先等待 5 到 10 分钟。
2. 打开 Tasks 页刷新。
3. 如果有 `task_id`，用 `hh_status` 查询。

### 12.5 成本确认弹窗

快乐马工作室会在超过预算阈值时弹出确认。确认前请注意：
