## 三、核心组件设计

### 3.1 触发器节点 (Trigger Nodes)

**视觉特征**: 绿色边框，圆角矩形，左侧带触发图标

```
┌─────────────────────────────┐
│ 🕐 定时触发                 │
│ ─────────────────────────── │
│ 每天 09:00                  │
│ 每个工作日的上午 9 点执行     │
└─────────────────────────────┘
```

**类型**:
1. **定时触发** -  Cron 表达式配置
2. **事件触发** - 系统事件监听
3. **Webhook** - API 端点接收
4. **手动触发** - 点击执行

### 3.2 动作节点 (Action Nodes)

**视觉特征**: 蓝色边框，圆角矩形，左侧带功能图标

```
┌─────────────────────────────┐
│ 📧 发送邮件                 │
│ ─────────────────────────── │
│ 收件人：{{customer_email}}  │
│ 主题：订单确认              │
└─────────────────────────────┘
```

**类型**:
1. **邮件处理** - 发送/接收/分类
2. **报表生成** - 数据汇总/图表
3. **数据整理** - 录入/清洗/转换
4. **API 调用** - HTTP 请求

### 3.3 条件分支节点 (Condition Node)

**视觉特征**: 黄色菱形，带 True/False 两个输出端口

```
         ┌──────────┐
         │  条件判断 │
         │ 金额>1000 │
         └────┬─────┘
         ╱         ╲
    True ╱           ╲ False
       ╱               ╲
```

### 3.4 循环节点 (Loop Node)

**视觉特征**: 紫色边框，带循环箭头图标

```
┌─────────────────────────────┐
│ 🔄 遍历订单列表             │
│ ─────────────────────────── │
│ 对每个订单项执行            │
│ 共 {{order_count}} 项       │
└─────────────────────────────┘
```

### 3.5 数据转换节点 (Transform Node)

**视觉特征**: 橙色边框，带数据转换图标

```
┌─────────────────────────────┐
│ 🔀 字段映射                 │
│ ─────────────────────────── │
│ order_id → 订单编号         │
│ amount → 金额               │
└─────────────────────────────┘
```

## 四、画布交互设计

### 4.1 节点操作

| 操作 | 交互方式 | 说明 |
|------|----------|------|
| 拖拽节点 | 鼠标左键按住拖动 | 从左侧面板拖入画布 |
| 移动节点 | 鼠标左键按住节点拖动 | 在画布内调整位置 |
| 连线 | 从输出端口拖到输入端口 | 建立节点间连接 |
| 删除节点 | 选中后按 Delete 或点击删除按钮 | 移除节点 |
| 复制节点 | Ctrl+C / Ctrl+V | 快速复制 |
| 多选 | Shift+ 点击或框选 | 批量操作 |

### 4.2 画布导航

| 操作 | 交互方式 |
|------|----------|
| 平移 | 鼠标右键按住拖动 或 空格 + 左键拖动 |
| 缩放 | 鼠标滚轮 或 Ctrl+ 滚轮 |
| 适应屏幕 | 双击空白处 或 Ctrl+0 |
| 重置视图 | 点击工具栏"适应屏幕"按钮 |

### 4.3 连接线样式

- **默认状态**: 灰色实线 (#9CA3AF), 2px
- **选中状态**: 蓝色实线 (#2563EB), 3px
- **数据流动**: 蓝色虚线动画，从左到右
- **错误状态**: 红色虚线 (#EF4444), 2px

│ 节点名称               ✕    │ ← 关闭按钮

6. **数字输入** - 次数、时间、阈值

## 七、交互说明文档

### 7.1 创建新工作流

**流程**:
1. 点击顶部工具栏"新建工作流"按钮
2. 弹出对话框，输入工作流名称和描述
3. 点击"创建"，进入空白画布
4. 从左侧组件库拖入第一个触发器节点
5. 配置触发器参数
6. 继续添加后续节点并连线
7. 点击右上角"保存"按钮

### 7.2 节点配置流程

**流程**:
1. 点击画布中的节点
2. 右侧自动展开配置面板
3. 修改配置项（实时预览）
4. 点击"保存"或点击画布空白处（自动保存）
5. 配置完成后，节点显示绿色对勾标记

### 7.3 连线操作

**流程**:
1. 鼠标悬停在节点的输出端口（右侧）
2. 端口高亮显示，出现"+"图标
3. 按住鼠标左键开始拖拽连线
4. 拖到目标节点的输入端口（左侧）
5. 端口高亮时松开鼠标，完成连接
6. 连线自动采用贝塞尔曲线，美观流畅

### 7.4 测试工作流

**流程**:
1. 点击右上角"测试"按钮
2. 弹出测试配置对话框
3. 选择测试数据或手动输入
4. 点击"开始测试"
5. 工作流逐节点执行，高亮显示当前节点
6. 执行完成后显示结果报告
7. 可查看每个节点的输入/输出数据

### 7.5 错误处理

**场景 1: 节点配置不完整**
- 节点边框显示红色警告
- 鼠标悬停显示具体错误信息
- 保存时弹出错误列表

**场景 2: 连接无效**
- 不允许连接（端口不高亮）
- 已连接的无效连线显示红色虚线
- 点击连线显示错误原因

**场景 3: 执行失败**
- 失败节点显示红色背景
- 自动重试（如配置）
- 发送错误通知（如配置）

## 八、响应式设计

### 8.1 断点定义

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| Desktop | ≥1440px | 三栏布局（完整） |
| Laptop | 1024-1439px | 三栏布局（侧边栏缩小） |
| Tablet | 768-1023px | 两栏布局（侧边栏可折叠） |
| Mobile | <768px | 单栏布局（仅画布，侧边栏抽屉式） |

### 8.2 移动端适配

- 左侧组件库 → 底部抽屉，点击按钮弹出
- 右侧配置面板 → 全屏覆盖，可关闭
- 画布操作 → 支持触摸手势（双指缩放、单指平移）
- 节点尺寸 → 适当放大，便于触摸操作

## 九、动画与过渡

### 9.1 微交互动画

| 场景 | 动画效果 | 时长 |
|------|----------|------|
| 节点悬停 | 轻微上浮 + 阴影加深 | 150ms |
| 节点选中 | 边框高亮 + 背景色变化 | 150ms |
| 连线创建 | 线条跟随鼠标绘制 | 实时 |
| 面板展开 | 从右侧滑入 | 300ms |
| 删除节点 | 缩小淡出 | 200ms |
| 数据流动 | 虚线从起点到终点动画 | 500ms |

### 9.2 缓动函数

- 标准过渡：`cubic-bezier(0.4, 0, 0.2, 1)`
- 弹性效果：`cubic-bezier(0.68, -0.55, 0.265, 1.55)`
- 快速响应：`cubic-bezier(0.25, 0.1, 0.25, 1)`

## 十、可访问性设计

### 10.1 键盘导航

| 按键 | 功能 |
|------|------|
| Tab | 在可聚焦元素间切换 |
| Enter | 确认/打开配置 |
| Delete | 删除选中节点 |
| Ctrl+C/V | 复制/粘贴 |
| Ctrl+Z/Y | 撤销/重做 |
| 方向键 | 移动选中节点 (10px 步进) |
| Shift+ 方向键 | 快速移动 (50px 步进) |

### 10.2 屏幕阅读器支持

- 所有图标按钮带 `aria-label`
- 节点状态变化触发 `aria-live` 通知
- 配置面板标题使用 `h2` 层级
- 错误信息关联 `aria-describedby`

### 10.3 对比度要求

- 文字与背景对比度 ≥ 4.5:1 (AA 标准)
- 大文字对比度 ≥ 3:1
- 焦点状态明显可见（3px 蓝色外框）

## 十一、性能优化

### 11.1 渲染优化

- 画布采用 Canvas/SVG 混合渲染
- 节点采用虚拟滚动（仅渲染可视区域）
- 连线采用简化路径计算
- 防抖处理（缩放/平移 100ms）

### 11.2 数据优化

- 工作流数据增量保存
- 自动保存间隔 30 秒
- 历史记录本地缓存（最近 50 步）
- 大型工作流分页加载节点
