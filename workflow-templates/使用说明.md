## 三、模板详解

### WF-01: 电商订单处理

**适用场景**: 电商零售企业自动处理订单

**触发条件**:
- ✅ 新订单创建
- ✅ 支付成功回调

**执行流程**:
1. 验证订单信息（订单系统）
2. 检查库存（WMS）
3. 生成发货单（ERP）
4. 通知物流公司（物流 API）
5. 发送客户通知（短信/邮件）

**集成系统**: 淘宝/天猫、京东、拼多多、顺丰快递、企业微信

**配置参数**:
```json
{
  "workflow_name": "电商订单处理",
  "trigger_type": "event",
  "timeout": 300,
  "retry_count": 3,
  "notify_on_complete": true
}
```

## 四、API 集成指南

### 4.1 集成方式

工作流引擎支持以下集成方式：

1. **HTTP API**: 通过 RESTful API 调用外部服务
2. **数据库直连**: 直接访问 MySQL/PostgreSQL
3. **消息队列**: 通过 Redis/RabbitMQ 异步通信
4. **Webhook**: 接收外部事件触发

### 4.2 配置示例

```python
# 配置 API 连接
from mvp.integrations.base import BaseAPI, APIMode

class DingTalkAPI(BaseAPI):
    def _call_real(self, **kwargs):
        # 真实 API 调用
        pass
    
    def _call_mock(self, **kwargs):
        # Mock 调用（测试用）
        pass

# 使用 Mock 模式测试
api = DingTalkAPI(mode=APIMode.MOCK)
```

## 五、常见问题

### Q1: 模板导入失败？

**原因**: 文件编码问题或模板格式错误

**解决**:
1. 确保模板文件为 UTF-8 编码
2. 检查模板是否包含必需字段（模板 ID、执行步骤）
3. 运行 `python mvp\workflow_engine.py` 查看详细错误日志

### Q2: 工作流执行超时？

**原因**: 步骤执行时间超过配置的 timeout

**解决**:
1. 增加 timeout 参数（默认 300 秒）
2. 检查集成的外部服务是否响应缓慢
3. 启用异步执行模式

### Q3: 如何自定义模板？

**步骤**:
1. 复制现有模板文件
2. 修改模板 ID、名称、执行步骤
3. 保存为 `.md` 文件到 `workflow-templates` 目录
4. 重新运行引擎加载新模板

### Q4: 如何集成自己的 API？

**步骤**:
1. 在 `mvp/integrations/` 目录创建 API 类
2. 继承 `BaseAPI` 基类
3. 实现 `_call_real` 和 `_call_mock` 方法
4. 在工作流步骤中调用
