## 一、产品概述

### 1.1 产品愿景
> "让 AI 真正帮你干活，而不只是聊天"

### 1.2 产品定位
面向 50-500 人中小企业的**执行型 AI Agent**，通过可视化工作流编排和预置行业模板，帮助企业自动化处理重复性工作，减少 50% 人力成本。

### 1.3 核心价值主张
| 价值点 | 描述 | 量化指标 |
|--------|------|----------|
| 效率提升 | 自动处理重复性工作 | 减少 50% 重复劳动 |
| 成本降低 | 1 个 Agent 替代 0.5 个人力 | 节省 50% 人力成本 |
| 快速上手 | 开箱即用，无需培训 | 3 分钟内完成首个任务配置 |
| 质量保证 | 标准化输出，减少人为错误 | 错误率<5% |
| 数据安全 | 支持私有化部署 | 数据完全自主可控 |

### 1.4 目标用户
| 用户类型 | 企业规模 | 典型行业 | 核心痛点 |
|----------|----------|----------|----------|
| 小型企业 | 50-100 人 | 电商、服务业 | 人力成本高，员工流动大 |
| 中型企业 | 100-300 人 | 制造业、专业服务 | 流程复杂，效率低下 |
| 大型企业 | 300-500 人 | 连锁零售、物流 | 数据孤岛，系统割裂 |

### 1.5 竞品差异化
| 维度 | 我们 | OpenAI Assistant | 国内大模型 Agent |

| 定位 | 执行型 Agent | 通用助手 | 通用助手 |

## 三、用户故事

### 3.1 电商行业用户故事

**US-001: 订单自动处理**
> 作为电商运营人员，我希望新订单产生时自动验证库存、生成发货单并通知仓库，以便减少人工操作错误并加快发货速度。

- **验收标准**:
  - [ ] 新订单产生后 1 分钟内触发工作流
  - [ ] 库存检查准确率 100%
  - [ ] 发货单自动生成并发送至仓库系统
  - [ ] 仓库人员收到通知（钉钉/企微）
  - [ ] 异常情况（库存不足）自动告警

**US-002: 客服邮件自动分类回复**
> 作为客服主管，我希望客户邮件自动分类（售前/售后/投诉）并匹配模板生成回复，以便减少 70% 重复性邮件处理工作。

- **验收标准**:
  - [ ] 邮件分类准确率>90%

  - [ ] 支持自定义回复模板
  - [ ] 邮件处理时间从 3 分钟降至 30 秒

**US-003: 销售日报自动生成**

## 四、UI 原型规范

### 4.1 工作流编排器 UI 规范

**设计原则**:
- 零代码，拖拽式操作
- 3 分钟上手，无需培训
- 上下文相关配置，减少认知负担

**核心界面元素**:

1. **左侧节点工具箱**
   - 触发器节点（蓝色）：定时/事件/API webhook
   - 动作节点（绿色）：邮件/报表/数据整理/API 调用
   - 条件节点（橙色）：if/else 分支
   - 循环节点（紫色）：for each/while
   - 工具节点（青色）：数据转换/字段映射

2. **中央画布区域**
   - 支持拖拽/连线/缩放/平移
   - 右键菜单（删除/复制/注释）
   - 网格吸附辅助对齐
   - 支持 100+ 节点，响应<2 秒

3. **右侧属性面板**
   - 选中节点时显示对应配置项
   - 表单式输入，支持验证
   - 提供"测试连接"按钮
   - 支持保存为模板

4. **顶部工具栏**
   - 保存/发布/运行/调试
   - 版本切换
   - 导入/导出
   - 帮助文档入口

**交互细节**:
- 拖拽节点时显示预览阴影
- 连线时自动吸附到节点连接点
- 删除节点前二次确认
- 支持 Ctrl+Z 撤销/重做
- 支持快捷键（Ctrl+S 保存，Ctrl+R 运行）

### 4.2 用户引导流程 UI 规范

**5 步上手路径**:

| 步骤 | 界面元素 | 预计时间 | 跳过机制 |
|------|----------|----------|----------|
| Step 1: 欢迎页 | 价值主张 +3 个案例展示 +CTA 按钮 | 30 秒 | 无跳过 |
| Step 2: 选择行业 | 4 个行业卡片 + 搜索框 | 30 秒 | "稍后选择" |
| Step 3: 模板预览 | 流程图可视化 + 节省时间显示 | 60 秒 | 无跳过 |
| Step 4: 关键配置 | 表单式配置 + 测试连接按钮 | 60 秒 | "稍后设置" |
| Step 5: 测试运行 | 执行动画 + 成功反馈 + 下一步建议 | 30 秒 | 无跳过 |

**关键设计细节**:
- 顶部进度条显示当前步骤（5 步）
- 每步右下角"？"帮助提示
- 支持返回上一步
- 跳过内容可在设置中补充
- 完成引导后显示 NPS 调研

### 4.3 数据看板 UI 规范

**核心指标卡片**:
- 今日执行次数（数字 + 趋势）
- 成功率（百分比 + 环比）
- 节省时间（小时 + 累计）
- 活跃工作流（数字 + 列表）

**图表组件**:
- 执行趋势图（折线图，7 天/30 天切换）
- 工作流排行（柱状图，Top 10）
- 错误分布（饼图，按错误类型）

**交互细节**:
- 支持时间范围选择（今日/本周/本月）
- 支持数据导出（Excel/PDF）
- 支持自定义看板（拖拽组件）
- 实时刷新（可配置频率）

## 六、技术实现方案

### 6.1 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端** | React 18 + TypeScript | 组件化开发，类型安全 |

### 6.3 核心模块设计

**工作流引擎**:
- 基于状态机的工作流执行
- 支持并行/串行/条件分支/循环
- 异步执行，支持长时间运行
- 错误重试机制（最多 3 次）
- 执行日志完整记录

**API 集成框架**:
- 统一适配器模式
- OAuth2/API Key 认证支持
- 请求限流和重试
- 响应数据标准化
- 支持 Webhook 回调

**模板系统**:
- JSON Schema 定义模板结构
- 支持模板变量和占位符
- 模板版本管理
- 一键导入/导出
- 模板市场（P2 功能）
