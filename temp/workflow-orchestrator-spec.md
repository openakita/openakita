## 三、核心功能模块

### 3.1 顶部导航栏

**功能**:
- 产品 Logo 和名称
- 预计配置时间计时器（实时显示用户已用时间）
- 重置按钮
- 运行工作流按钮

**交互**:
- 计时器从 0 开始，每添加一个节点 +15 秒
- 使用模板后显示模板预计时间（如 120 秒）
- 运行按钮在工作流节点≥1 时可点击

---

### 3.2 步骤指示器

**功能**: 引导用户完成工作流创建

**步骤**:
1. 选择一个行业模板
2. 拖拽节点到工作区
3. 配置节点参数
4. 测试并运行

**交互**:
- 当前步骤高亮显示（蓝色）
- 已完成步骤保持蓝色
- 未完成步骤显示灰色

---

### 3.3 左侧组件库

**包含 5 种节点类型**:

| 节点类型 | 图标 | 颜色 | 用途 |
|----------|------|------|------|
| 🔔 触发器 | 🔔 | 绿色 | 工作流启动条件（定时/事件/API） |
| ⚡ 执行动作 | ⚡ | 蓝色 | 具体执行任务（发送邮件/生成报表等） |
| 🔀 条件分支 | 🔀 | 黄色 | 逻辑判断（是/否分支） |
| 🔄 循环 | 🔄 | 紫色 | 批量处理（遍历数据） |
| ⚠️ 异常处理 | ⚠️ | 红色 | 错误捕获和处理 |

**交互**:
- 所有节点支持拖拽
- 拖拽时显示"copy"光标
- 拖拽到工作区后生成新节点

---

### 3.4 行业模板选择区

**10 个预置模板**（覆盖 4 大行业）:

#### 电商行业（3 个）
1. 📦 订单自动处理（2 分钟）
2. 📧 客服邮件分类回复（3 分钟）
3. 📊 销售日报自动生成（2 分钟）

#### 服务业（3 个）
4. 📅 客户预约自动确认（2 分钟）
5. 🔧 服务工单自动分配（3 分钟）
6. ⭐ 客户满意度调研（2 分钟）

#### 制造业（2 个）
7. 📦 库存预警补货（3 分钟）
8. 🏭 生产日报汇总（2 分钟）

#### 专业服务（2 个）
9. 📄 合同到期提醒（2 分钟）
10. 📝 会议纪要待办（3 分钟）

**交互**:
- 点击模板卡片即加载预设工作流
- 加载后步骤指示器自动跳到第 2 步
- 显示模板预计配置时间

---

### 3.5 中央工作区

**功能**: 拖拽式工作流设计画布

**状态**:
1. **空状态**: 显示引导文案和图标，提示用户拖拽或选择模板
2. **有内容**: 显示节点列表，节点间用连接线表示流程

**节点展示**:
- 左侧彩色边框标识节点类型（绿/蓝/黄/紫/红）
- 节点图标 + 名称 + 类型标签
- 悬停时显示"编辑"和"删除"按钮
- 节点间用垂直线连接，表示执行顺序

**交互**:
- 支持拖拽节点进入
- 拖拽进入时显示蓝色高亮边框
- 节点可删除、可编辑
- 自动计算节点数量

---

### 3.6 右侧属性面板

**功能**: 配置选中节点的详细参数

**配置项**:
1. **基础配置**
   - 节点名称（文本输入）
   - 触发条件（下拉选择：新数据/定时/手动/API）
   - 执行动作（下拉选择：邮件/报表/数据库/API/通知）

2. **高级设置**
   - 启用异常处理（复选框）
   - 记录执行日志（复选框）

3. **执行统计**
   - 预计节省时间（根据节点数计算）
   - 工作流复杂度评估（未开始/简单/中等）

**交互**:
- 无节点时显示提示文案
- 有节点时显示配置表单
- 所有配置项实时更新统计信息

## 五、差异化设计亮点

### 5.1 对比竞品的优势

| 功能 | 我们 | OpenAI Assistant | 国内大模型 Agent |
|------|------|------------------|------------------|
| 上手时间 | **3 分钟** | 需学习（10+ 分钟） | 需学习（10+ 分钟） |
| 配置方式 | **拖拽式，无需代码** | 代码/配置混合 | 配置为主 |
| 行业模板 | **10 个预置模板** | 无 | 少量 |
| 可视化程度 | **完整流程图** | 弱 | 中 |
| 异常处理 | **可视化配置** | 需代码 | 不支持 |

### 5.2 设计原则

1. **3 分钟上手**: 通过模板 + 引导，确保新用户 3 分钟内完成首个任务
2. **零代码**: 所有配置通过表单和下拉选择，无需编写代码
3. **可视化**: 完整流程图展示，执行顺序一目了然
4. **容错性**: 内置异常处理节点，降低使用门槛

## 六、技术实现建议

### 6.1 前端技术栈

**推荐**:
- React 18 + TypeScript
- React Flow / X6（流程图引擎）
- Tailwind CSS + shadcn/ui
- Zustand（状态管理）

### 6.2 核心组件

```
WorkflowEditor/
├── ComponentLibrary/      # 左侧组件库
├── TemplateGallery/       # 模板选择区
├── WorkflowCanvas/        # 中央画布
│   ├── WorkflowNode/     # 节点组件
│   ├── ConnectionLine/   # 连接线
│   └── DropZone/         # 拖拽区域
├── PropertyPanel/         # 右侧属性面板
└── StepIndicator/         # 步骤指示器
```

### 6.3 数据结构

```typescript
interface Workflow {
  id: string;
  name: string;
  templateId?: string;
  nodes: WorkflowNode[];
  connections: Connection[];
  createdAt: Date;
  updatedAt: Date;
}

interface WorkflowNode {
  id: string;
  type: 'trigger' | 'action' | 'condition' | 'loop' | 'exception';
  name: string;
  config: Record<string, any>;
  position: { x: number; y: number };
}

interface Connection {
  id: string;
  sourceNodeId: string;
  targetNodeId: string;
  sourceHandle?: string; // 条件分支的"是"/"否"
  targetHandle?: string;
}
```
