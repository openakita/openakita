## 一、目标客户画像

### 1.1 客户分层标准

| 维度 | 标准 | 说明 |
|------|------|------|
| **企业规模** | 50-500 人 | 中型企业，有数字化预算但不足以定制开发 |
| **年营收** | 5000 万 -10 亿元 | 有足够利润空间投资效率工具 |
| **数字化程度** | 已有基础系统 (ERP/CRM/OA) | 需要系统间自动化，而非从零建设 |
| **决策链条** | 老板/CTO/运营总监 | 2-3 人决策，周期 2-4 周 |
| **付费意愿** | ¥5000-50000/月 | 认可 ROI，愿意为效率付费 |

### 1.2 重点行业画像

#### 行业 A: 电商企业 (优先级 P0)

**典型特征**:
- 规模：50-200 人团队，年 GMV 5000 万 -5 亿元
- 系统：电商 ERP+ 客服系统+WMS+ 财务软件
- 痛点：
  - 订单处理：多平台订单手动汇总，易出错 (90% 企业痛点)
  - 库存同步：各平台库存不同步，超卖/缺货频发 (85% 企业痛点)
  - 客服回复：重复问题占比 80%，人工成本高 (80% 企业痛点)
  - 数据报表：每日手动整理各平台数据，耗时 2-3 小时 (75% 企业痛点)

**决策人关注点**:
- 老板：ROI、成本节省、竞争优势
- 运营总监：效率提升、错误率降低、团队减负
- IT 负责人：系统稳定性、集成难度、数据安全

**典型场景**:
```
场景 1: 订单自动处理
触发：新订单产生 (淘宝/京东/拼多多)
→ 自动同步到 ERP
→ 自动分配仓库
→ 自动推送物流单号
→ 自动发送客户通知
节省：3 小时/天 → 3 分钟/天

场景 2: 库存自动同步
触发：某平台库存变化
→ 自动同步所有平台
→ 低于阈值自动预警
→ 自动生成采购建议
节省：2 小时/天 → 实时自动
```

**付费意愿**: ¥10000-30000/月 (按订单量阶梯定价)

---

#### 行业 B: 服务业 (优先级 P0)

**典型特征**:
- 规模：100-500 人，连锁门店/服务机构
- 系统：CRM+ 预约系统 + 财务软件+OA
- 痛点：
  - 客户跟进：销售线索分配不及时，跟进记录分散 (85% 企业痛点)
  - 预约管理：人工排班易冲突，提醒不到位 (70% 企业痛点)
  - 报表统计：各门店数据手动汇总，时效性差 (80% 企业痛点)
  - 审批流程：纸质/微信审批，流程不透明 (65% 企业痛点)

**决策人关注点**:
- 老板：标准化、可复制、管理半径
- 运营总监：流程规范、数据透明、人效提升
- 门店店长：操作简单、减少事务性工作

**典型场景**:
```
场景 1: 销售线索自动分配
触发：新线索进入 CRM
→ 自动分配给对应区域销售
→ 自动发送跟进提醒
→ 超时未跟进自动升级
→ 自动记录跟进结果
节省：1 小时/天 → 实时自动

场景 2: 预约自动管理
触发：客户预约
→ 自动检查技师/房间空闲
→ 自动发送确认短信
→ 提前 1 天自动提醒
→ 爽约自动标记并释放资源
节省：2 小时/天 → 全自动
```

**付费意愿**: ¥8000-25000/月 (按门店数/人数定价)

---

#### 行业 C: 制造业 (优先级 P1)

**典型特征**:
- 规模：100-500 人，离散制造为主
- 系统：ERP+MES+ 采购系统 + 仓储系统
- 痛点：
  - 生产排期：人工排产，调整频繁，响应慢 (80% 企业痛点)
  - 物料管理：库存不准，缺料/积压并存 (75% 企业痛点)
  - 质量追溯：纸质记录，追溯困难 (70% 企业痛点)
  - 设备维护：被动维修，停机损失大 (65% 企业痛点)

**决策人关注点**:
- 老板：产能利用率、交付准时率、成本管控
- 生产总监：排产效率、异常响应、质量稳定
- IT 负责人：系统兼容、数据安全、私有化部署

**典型场景**:
```
场景 1: 生产订单自动排程
触发：新订单进入 ERP
→ 自动检查物料库存
→ 自动计算产能负荷
→ 自动生成排产计划
→ 自动推送车间执行
节省：4 小时/天 → 10 分钟/天

场景 2: 物料自动预警
触发：物料库存低于安全线
→ 自动生成采购申请
→ 自动推送采购部
→ 自动跟踪到货进度
→ 自动更新库存预期
节省：1 小时/天 → 实时自动
```

**付费意愿**: ¥15000-50000/月 (按产线/用户数定价)

### 3.2 四大差异化卖点

#### 卖点 1: 私有化部署 (竞品弱项)

**话术**:
> "数据不出企业，100% 自主可控"

**支撑点**:
- 支持本地/私有云部署
- 数据完全隔离，不与第三方共享
- 符合等保 2.0/数据安全法要求
- 金融行业已验证案例

**对比竞品**:
- OpenAI: 仅云端，数据需上传
- Zapier: 仅云端，数据经第三方
- 国内大模型：部分支持，但功能受限

**目标客户收益**:
- 数据安全：避免商业机密泄露
- 合规要求：满足行业监管
- 性能可控：内网部署，响应更快

---

#### 卖点 2: 行业模板库 (竞品无)

**话术**:
> "50+ 行业模板，3 分钟上手，1 小时见效"

**支撑点**:
- 预置电商/服务业/制造业 50+ 标准工作流
- 一键导入，无需从零配置
- 持续更新，跟随最佳实践
- 支持自定义扩展

**模板示例**:
- 电商：订单自动处理、库存同步、客服自动回复
- 服务业：线索自动分配、预约管理、报表自动汇总
- 制造业：生产排程、物料预警、质量追溯

**对比竞品**:
- OpenAI: 无行业模板，需自行开发
- Zapier: 通用模板为主，缺乏行业深度
- 国内大模型：少量模板，更新慢

**目标客户收益**:
- 快速上线：从数周缩短到数小时
- 降低门槛：非技术人员也可配置
- 最佳实践：直接复用行业经验

---

#### 卖点 3: 3 分钟上手 (竞品弱项)

**话术**:
> "无需培训，3 分钟完成首个自动化任务"

**支撑点**:
- 拖拽式工作流编辑器
- 可视化配置，无需代码
- 智能引导，新手友好
- 内置教程 + 在线支持

**用户体验流程**:
```
注册账号 (1 分钟)
→ 选择行业模板 (30 秒)
→ 配置触发器和操作 (1 分钟)
→ 测试运行 (30 秒)
→ 正式上线
总计：3 分钟
```

- OpenAI: 需学习 Prompt 工程，上手 1-2 小时
- Zapier: 需理解触发器/动作逻辑，上手 30 分钟
- 国内大模型：功能分散，学习成本 1 小时+

**目标客户收益**:
- 降低培训成本：无需专门培训

- 全员可用：非技术人员也能使用
