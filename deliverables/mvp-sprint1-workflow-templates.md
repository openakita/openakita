## 三、技术实现

### 3.1 工作流引擎架构

```
WorkflowEngine
├── 模板加载器 (_load_templates)
├── 模板解析器 (_parse_template)
├── 执行引擎 (execute)
├── 实例管理 (instances)
└── API 集成层 (integrations/)
```

### 3.2 核心功能

1. **模板解析**: 自动解析 Markdown 格式模板，提取步骤/触发条件/集成系统
2. **工作流执行**: 支持同步/异步执行，带状态追踪
3. **实例管理**: 每个工作流实例有独立 ID 和状态
4. **异常处理**: 失败重试、超时控制、错误日志
5. **API 集成**: 统一的 API 抽象层（Mock/Real 双模式）

## 四、使用示例

### 4.1 快速导入

```bash
cd D:\coder\myagent
python mvp\workflow_engine.py
```

### 4.2 编程调用

```python
from mvp.workflow_engine import WorkflowEngine

# 初始化引擎
engine = WorkflowEngine(templates_dir="workflow-templates")

# 列出所有模板
templates = engine.list_templates()
for t in templates:
    print(f"{t['template_id']}: {t['name']}")

# 执行工作流
instance = engine.execute("WF-01")
print(f"状态：{instance.status.value}")
print(f"结果：{instance.result}")
```
