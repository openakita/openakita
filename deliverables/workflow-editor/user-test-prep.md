### 1.2 测试任务设计（初稿）

**任务 1: 创建订单处理工作流**（电商）
- 场景：你是电商运营，需要自动处理新订单
- 操作：导入"订单自动处理"模板，配置 API 密钥，测试运行
- 预期时间：<3 分钟
- 成功标准：工作流成功创建并测试通过

**任务 2: 创建邮件分类工作流**（服务业）
- 场景：你是客服主管，需要自动分类客户邮件
- 操作：导入"客服邮件自动分类"模板，配置邮箱，设置分类规则
- 预期时间：<3 分钟
- 成功标准：邮件分类规则配置完成

**任务 3: 自定义简单工作流**（通用）
- 场景：你需要创建一个简单的日报发送工作流
- 操作：从空白开始，添加定时触发器，添加邮件动作，连接并配置
- 预期时间：<5 分钟
- 成功标准：工作流成功创建

**任务 4: 查看执行历史**（通用）
- 场景：你想查看昨天工作流的执行情况
- 操作：找到执行历史页面，查看某个工作的执行详情
- 预期时间：<1 分钟
- 成功标准：成功找到并查看执行详情

**任务 5: 邀请团队成员**（通用）
- 场景：你想邀请同事一起使用这个工具
- 操作：找到团队管理页面，发送邀请链接
- 预期时间：<2 分钟
- 成功标准：成功发送邀请
