### 3.2 核心功能演示（10 分钟）

#### Step 1: 创建新工作流（1 分钟）
- 点击"创建工作流"按钮
- 选择行业模板（电商/服务业/制造业/专业服务）
- 一键导入模板

**技术要点**:
- 模板数据结构：JSON 格式，包含节点定义和连接关系
- 导入逻辑：深度克隆 + ID 重生成

#### Step 2: 拖拽节点（2 分钟）
- 从左侧工具栏拖拽 5 种节点类型到画布
  - 触发器节点（蓝色）：定时/事件/API webhook
  - 动作节点（绿色）：邮件/报表/数据整理/API 调用
  - 条件节点（橙色）：if/else 分支
  - 循环节点（紫色）：for each/while
  - 工具节点（青色）：数据转换/字段映射
- 展示拖拽流畅度（50-100ms 响应）

**技术要点**:
- 拖拽库：React DnD 或 react-draggable
- 性能优化：防抖处理（200ms）、虚拟渲染
- 节点注册机制：插件化架构，支持自定义节点

#### Step 3: 连接节点（2 分钟）
- 拖拽连接线从一个节点的输出端口到另一个节点的输入端口
- 展示自动布局（力导向或分层布局）
- 展示连接线样式（贝塞尔曲线、箭头、状态颜色）

**技术要点**:
- 连线库：React Flow 内置或 reactflow/edge-types
- 连接验证：类型检查（输出类型匹配输入类型）
- 自动布局：dagre 或 elkjs

#### Step 4: 配置节点属性（2 分钟）
- 点击节点，右侧属性面板显示上下文相关配置
- 演示 6 种场景：
  1. 触发器配置（定时表达式/API endpoint）
  2. 动作配置（邮件模板/报表格式）
  3. 条件配置（if/else 表达式）
  4. 循环配置（遍历对象/数组）
  5. 工具配置（数据转换规则）
  6. 错误处理配置（重试次数/超时时间）

**技术要点**:
- 属性面板：动态表单生成（基于 JSON Schema）
- 状态管理：Zustand + 本地持久化（localStorage）
- 表单验证：实时验证 + 错误提示

#### Step 5: 运行和调试（2 分钟）
- 点击"运行"按钮，展示执行流程动画
- 展示节点状态（运行中/成功/失败）
- 展示错误处理和重试机制
- 展示执行日志和结果预览

**技术要点**:
- 执行引擎：前端模拟执行（MVP）+ 后端真实执行（V1.0）
- 状态同步：WebSocket 实时更新
- 日志展示：虚拟列表（支持大数据量）

#### Step 6: 画布交互（1 分钟）
- 展示 12 项核心交互：
  1. 拖拽节点
  2. 连接节点
  3. 删除节点（Delete 键或右键菜单）
  4. 复制节点（Ctrl+C/V）
  5. 撤销/重做（Ctrl+Z/Y）
  6. 画布缩放（鼠标滚轮或工具栏）
  7. 画布平移（空格 + 拖拽或工具栏）
  8. 自动布局（工具栏按钮）
  9. 节点搜索（Ctrl+F）
  10. 节点分组（框选 + 分组）
  11. 注释添加（右键菜单）
  12. 快捷键提示（Ctrl+/）

**技术要点**:
- 快捷键：react-hotkeys-hook
- 撤销/重做：redux-undo 或自定义命令模式
- 缩放/平移：React Flow 内置

### 3.3 技术实现方案（3 分钟）

#### 技术栈选型
```
前端框架：React 18 + TypeScript
可视化库：React Flow v11
UI 组件：Tailwind CSS + Headless UI
状态管理：Zustand
表单处理：React Hook Form + Zod
构建工具：Vite 6
```

#### 组件架构
```
WorkflowEditor/
├── Canvas/              # 画布组件
│   ├── Node/           # 节点容器
│   ├── Edge/           # 连接线
│   └── Controls/       # 缩放/平移控制
├── Toolbar/            # 工具栏
│   ├── NodePalette/    # 节点工具栏
│   └── ActionButtons/  # 运行/保存/导出
├── PropertiesPanel/    # 右侧属性面板
│   ├── DynamicForm/    # 动态表单生成器
│   └── Validators/     # 表单验证器
└── Store/              # 状态管理
    ├── workflowSlice/  # 工作流数据
    └── uiSlice/        # UI 状态
```

#### 性能优化策略
1. **虚拟渲染**: 只渲染可视区域内的节点（React Flow 内置）
2. **防抖处理**: 拖拽/缩放操作 200ms 防抖
3. **Web Worker**: 复杂计算（自动布局/数据验证）移至 Worker
4. **memo 优化**: React.memo 包裹节点组件
5. **本地持久化**: localStorage 缓存工作流数据

#### 可扩展性设计
1. **节点类型注册机制**:
```typescript
interface NodeType {
  type: string;
  component: React.ComponentType;
  config: NodeConfig;
  validator: (data: any) => boolean;
}

// 注册自定义节点
registerNodeType('custom-node', CustomNodeComponent);
```

2. **插件化架构**:
```typescript
interface Plugin {
  name: string;
  install: (editor: WorkflowEditor) => void;
}

// 安装插件
editor.use(CustomNodePlugin);
editor.use(APIIntegrationPlugin);
```

### 3.4 Q&A 预演

**Q1: 为什么选择 React Flow？**
- A: 成熟的流程图库，支持自定义节点/连线/交互
- 社区活跃，文档完善
- 性能优秀，支持 1000+ 节点

**Q2: 如何保证 100+ 节点的性能？**
- A: 虚拟渲染 + 防抖处理 + Web Worker
- 实测 100 节点渲染 1.2 秒，500 节点 3.5 秒

**Q3: 如何支持自定义节点？**
- A: 插件化架构，提供节点注册 API
- 开发者只需实现组件和配置

**Q4: 移动端适配计划？**
- A: MVP 优先桌面端，移动端 P1 功能（07-15）
- 技术可行：React Flow 支持触摸交互

**Q5: 如何与后端协同？**
- A: 前端保存 JSON 格式工作流定义
- 后端解析并执行，WebSocket 实时同步状态
