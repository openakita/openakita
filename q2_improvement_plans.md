### 二、现状分析

**当前性能指标**:
| 指标 | 当前值 | 行业基准 | 差距 |
|------|--------|----------|------|
| API P95 延迟 | 890ms | <500ms | -44% |
| 向量检索耗时 | 120ms | <30ms | -300% |
| LLM 缓存命中率 | 35% | >60% | -42% |
| 技能加载时间 | 450ms | <100ms | -350% |

**核心瓶颈**:
1. **向量数据库**: SQLite 向量搜索无索引，全表扫描

4. **数据库查询**: 部分慢查询缺少索引

### 三、技术方案

#### 3.1 向量数据库迁移 (Qdrant)

**技术选型**: Qdrant (Rust 实现，高性能向量数据库)

**迁移方案**:
```
阶段 1: 部署 Qdrant (Docker)
  - 本地开发：Docker Compose
  - 生产环境：Qdrant Cloud ($30/月)

阶段 2: 数据迁移
  - 导出 SQLite 向量数据
  - 批量导入 Qdrant
  - 验证数据完整性

阶段 3: 代码适配
  - 抽象向量存储接口
  - 实现 Qdrant 适配器
  - 灰度切换 (10% → 50% → 100%)

阶段 4: 性能验证
  - 基准测试对比
  - 生产环境监控
```

**预期收益**:
- 向量检索：120ms → 24ms (-80%)
- 并发查询：100 QPS → 800 QPS (+700%)
- 内存占用：-40% (Qdrant 压缩算法)

**工作量**: 16 工时

#### 3.2 技能加载缓存优化

**当前问题**: 每次技能执行重新解析 SKILL.md，重复 I/O

**优化方案**:
```python
# 三级缓存架构
L1: 内存缓存 (TTL 5 分钟) - 热点技能
L2: Redis 缓存 (TTL 1 小时) - 常用技能
L3: 磁盘缓存 (TTL 24 小时) - 全部技能
```

**实现细节**:
- 技能加载时计算内容哈希
- 哈希未变化则跳过解析
- 支持手动刷新缓存 (CLI 命令)

**预期收益**:
- 技能加载：450ms → 50ms (-89%)
- 技能执行 P99: 3.5s → 2.1s (-40%)

**工作量**: 4 工时

#### 3.3 LLM 响应缓存策略优化

**当前问题**: 缓存 key 设计简单，命中率仅 35%

**优化方案**:
```python
# 智能缓存 key 生成
cache_key = hash(
    prompt_template_id +      # 提示词模板 ID
    user_intent_hash +        # 用户意图哈希 (Embedding 聚类)
    context_window_hash +     # 上下文窗口哈希
    temperature +             # 温度参数
    max_tokens                # 最大 token 数
)

# 语义缓存 (Semantic Caching)
- 使用 Embedding 计算问题相似度
- 相似度 >0.95 视为相同问题
- 返回缓存结果，跳过 LLM 调用
```

**预期收益**:
- LLM 缓存命中率：35% → 60% (+71%)
- LLM 调用成本：-25%
- 平均响应时间：-15%

**工作量**: 8 工时

**加密范围**:

- 聊天记录
- 向量嵌入
- 用户画像数据

**加密方案**:
```python
# AES-256-GCM 加密
from cryptography.fernet import Fernet

# 密钥管理
- 主密钥：KMS 托管 (AWS KMS / 阿里云 KMS)
- 数据密钥：每用户独立，主密钥加密存储
- 密钥轮换：90 天自动轮换

# 加密流程
1. 生成随机数据密钥 (DEK)
2. 使用 DEK 加密数据
3. 使用主密钥 (KEK) 加密 DEK
4. 存储加密数据 + 加密 DEK
```

**预期收益**:
- 数据加密覆盖率：0% → 100%

#### 3.2 JWT Token 轮换机制

**当前问题**: JWT Token 长期有效，泄露风险高

**轮换方案**:
```
Access Token (短期):
- 有效期：15 分钟
- 用途：API 调用认证
- 存储：内存 (不持久化)

Refresh Token (长期):
- 有效期：7 天
- 用途：获取新 Access Token
- 存储：httpOnly Cookie (加密)
- 轮换：每次使用后立即失效，生成新 Token

Token 绑定:
- 绑定设备指纹
- 绑定 IP 地址段
- 异常检测：异地登录告警
```

**预期收益**:
- Token 泄露风险：-80%
- 会话劫持风险：-90%

**工作量**: 8 工时
