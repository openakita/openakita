## 🔧 扩展策略

### 垂直扩展（Scale Up）

**何时使用**: 单机资源不足，但用户量增长可预测

**扩展路径**:
```
4 核 8GB → 8 核 16GB → 16 核 32GB → 32 核 64GB
```

**优点**:
- 简单，无需修改架构
- 成本低（初期）
- 运维复杂度低

**缺点**:
- 有上限（硬件限制）
- 单点故障风险
- 停机升级

---

### 水平扩展（Scale Out）

**何时使用**: 高并发、高可用需求

**扩展组件**:

#### 1. 应用服务（无状态）

```yaml
# docker-compose 扩展
services:
  app:
    deploy:
      replicas: 3  # 3 个副本
      resources:
        limits:
          cpus: '2.0'
          memory: 2G
```

**负载均衡**: Nginx / HAProxy / AWS ALB

---

#### 2. Celery Worker（无状态）

```bash
# 增加 Worker 数量
docker-compose up -d --scale celery-worker=4
```

**队列优化**:
- 按优先级分队列（high/default/low）
- 按任务类型分队列（llm/browser/file）

---

#### 3. PostgreSQL（有状态）

**主从复制**:
```
Master (读写) → Slave1 (只读) → Slave2 (只读)
```

**读写分离**:
- 写操作：Master
- 读操作：Slave（负载均衡）

**分库分表**（百万级数据）:
- 按用户 ID 哈希分库
- 按时间范围分表（会话/消息）

---

#### 4. Redis（有状态）

**Redis Cluster**:
```
3 Master + 3 Slave (6 节点)
```

**数据分片**:
- 16384 个 hash slot
- 自动故障转移

---

#### 5. Qdrant（有状态）

**分布式集群**:
```
3 节点集群（Raft 共识）
```

**分片策略**:
- 自动分片（按向量 ID）
- 副本因子：2

---

### 自动扩展（Auto Scaling）

**云原生方案**:

```yaml
# Kubernetes HPA
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: openakita-app
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: openakita-app
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

**触发条件**:
- CPU 使用率 > 70%
- 内存使用率 > 80%
- 请求队列长度 > 100

**冷却时间**: 300 秒（防止抖动）

## 📊 容量规划工具

### 用户量估算公式

```
所需 QPS = (日活用户 × 人均请求数) / (8 小时 × 3600 秒)

示例:
日活 1000 用户，人均 50 次请求
所需 QPS = (1000 × 50) / (8 × 3600) = 1.74 QPS

考虑峰值系数 3:
峰值 QPS = 1.74 × 3 = 5.22 QPS
```

---

### 存储增长预测

```
年存储增量 = (日增量 × 365) × (1 + 年增长率)

示例:
日增量 200 MB，年增长率 50%
年存储增量 = (200 MB × 365) × 1.5 = 109.5 GB

3 年总存储 = 初始存储 + 年增量 × 3
           = 50 GB + 109.5 GB × 3 = 378.5 GB
```

---

### 带宽需求估算

```
带宽 (Mbps) = (QPS × 平均响应大小 × 8) / 1,000,000

示例:
QPS=100，平均响应 10 KB
带宽 = (100 × 10 KB × 8) / 1,000,000 = 8 Mbps

考虑峰值系数 3:
峰值带宽 = 8 × 3 = 24 Mbps
```
