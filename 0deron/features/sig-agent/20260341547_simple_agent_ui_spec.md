# 非任务场景 UI 设计规范（普通聊天 + 单 Agent 步骤）

> 适用范围：SeeAgent WebUI 中 **不涉及最佳实践/多步骤任务** 的场景，即日常对话、单 Agent 工具调用。
> 本文档仅覆盖非任务场景，任务场景（TaskBoard、SubAgent 步骤等）另文描述。

## 2. 机器人回复结构

### 2.1 回复头部：头像 + 计时器

每条机器人回复的第一行统一结构：

```
┌─────────────────────────────────────────────────┐
│  🤖 OpenAkita          TTFT: 1.2s  Total: 4.8s  │
└─────────────────────────────────────────────────┘
```

#### 计时器说明

| 计时器  | 含义                          | 运行态              | 完成态           |
|--------|-------------------------------|--------------------|--------------------|
| TTFT   | 用户发送 → 首个 token 到达      | 蓝色脉冲圆点 + 计时中 | 灰色静态最终值     |
| Total  | 用户发送 → 回复全部完成          | 蓝色脉冲圆点 + 计时中 | 灰色静态最终值     |

运行态视觉示例：
```
  ● TTFT: 0.8s   ● Total: 2.3s...        (蓝色脉冲 + 数字递增)
```

完成态视觉示例：
```
  ○ TTFT: 1.2s   ○ Total: 4.8s           (灰色静态)
```

### 2.2 普通模式（Normal Mode）—— 三段式结构

适用于简单问答、0~2 个工具调用的场景。

```
┌─────────────────────────────────────────────────┐
│  🤖 OpenAkita           TTFT: 1.2s  Total: 4.8s │
├─────────────────────────────────────────────────┤
│                                                  │
│  ▶ Thinking...                             ↓    │  ← 折叠区，点击 ↓ 展开
│                                                  │
│  ┌─────────────────────────────────────────┐    │
│  │ ● 搜索 "Karpathy 2026 最新观点"      →  │    │  ← 步骤卡片（条状）
│  └─────────────────────────────────────────┘    │
│                                                  │
│  根据搜索结果，Karpathy 最近在 2026 年 3 月    │  ← 总结输出
│  发表了关于 AI Agent 的最新看法...              │
│  巴拉巴拉一段总结                               │
│                                                  │
└─────────────────────────────────────────────────┘
```

三段式组成：

```
  ┌──────────────────────┐
  │  1. Thinking 折叠区   │   默认折叠，点击 ↓ 展开查看推理过程
  ├──────────────────────┤
  │  2. Step Cards       │   0~N 个条状步骤卡片（可能没有）
  ├──────────────────────┤
  │  3. Summary Output   │   最终文本输出（markdown 渲染）
  └──────────────────────┘
```

### 2.3 计划模式（Plan Mode）—— 四段式结构

当 Agent 自主判断当前任务需要多步执行时，自动切换到计划模式。

```
┌──────────────────────────────────────────────────────┐
│  🤖 OpenAkita              TTFT: 0.9s  Total: 12.3s  │
├──────────────────────────────────────────────────────┤
│                                                       │
│  ▶ Thinking...                                  ↓   │  ← 折叠区
│                                                       │
│  ┌─────────────────────────────────────────────┐     │
│  │  执行计划                                    │     │  ← Plan Checklist
│  │  ☑ 1. 搜索 Karpathy 最新观点                 │     │     独立展开区块
│  │  ☑ 2. 整理要点并归类                         │     │     默认展开
│  │  ☐ 3. 生成 PDF 报告              (进行中)    │     │
│  │  ☐ 4. 输出最终摘要                           │     │
│  └─────────────────────────────────────────────┘     │
│                                                       │
│  ┌─────────────────────────────────────────────┐     │
│  │ ✓ 搜索 "Karpathy 2026"                  →  │     │  ← 步骤卡片
│  └─────────────────────────────────────────────┘     │
│  ┌─────────────────────────────────────────────┐     │
│  │ ✓ 整理要点并归类                          →  │     │
│  └─────────────────────────────────────────────┘     │
│  ┌─────────────────────────────────────────────┐     │
│  │ ● 生成 PDF 报告                          →  │     │  ← 当前执行中
│  └─────────────────────────────────────────────┘     │
│                                                       │
│  (等待所有步骤完成后输出总结...)                       │
│                                                       │
└──────────────────────────────────────────────────────┘
```

四段式组成：

```
  ┌──────────────────────────┐
  │  1. Thinking 折叠区       │   默认折叠
  ├──────────────────────────┤
  │  2. Plan Checklist       │   独立区块，默认展开，非 Thinking 内部
  ├──────────────────────────┤
  │  3. Step Cards           │   逐步出现的条状卡片
  ├──────────────────────────┤
  │  4. Summary Output       │   全部完成后输出总结
  └──────────────────────────┘
```

Plan Checklist 行为规则：

| 规则                  | 说明                                         |
|----------------------|----------------------------------------------|
| 位置                  | Thinking 折叠区下方，Step Cards 上方，独立区块    |
| 默认状态              | 展开                                          |
| Checkbox 更新         | 对应步骤完成时自动勾选 ☐ → ☑                    |
| 当前步骤标记           | 显示 "(进行中)" 文字提示                        |
| 可折叠                | 支持用户手动折叠/展开                            |

## 3. 两种模式对比

### 3.1 对比表

| 维度           | 普通模式 (Normal)         | 计划模式 (Plan)                     |
|---------------|--------------------------|-------------------------------------|
| 触发方式       | 默认模式                  | Agent 自主判断触发（多步复杂任务）     |
| 回复结构       | 三段式                    | 四段式（多一个 Plan Checklist）       |
| 典型步骤数     | 0~2 个                   | 3 个以上                             |
| Plan Checklist | 无                       | 有，独立展开区块                      |
| Thinking       | 折叠，简短                | 折叠，可能包含计划推理                 |
| Summary Output | 直接输出                  | 等全部步骤完成后输出                   |
| 适用场景       | 简单问答、单次搜索、翻译   | 多步研究、报告生成、数据分析流程       |

### 3.2 模式判定（后端逻辑）

```
用户消息 → Agent 分析
  ├─ 判定为简单任务 → Normal Mode（三段式）
  └─ 判定为复杂多步 → Plan Mode（四段式）
       └─ 先输出 plan_checklist 消息
       └─ 再逐步执行并输出 step_card
```

> 前端不做模式判断。前端根据收到的消息类型自适应渲染：
> 收到 `plan_checklist` 类型消息 → 渲染四段式；未收到 → 渲染三段式。

## 5. 步骤过滤规则（后端负责）

### 5.1 过滤原则

前端展示的步骤卡片并非 Agent 执行的所有工具调用，后端负责过滤，只下发**用户可理解的语义步骤**。

### 5.2 展示 vs 隐藏

| 类型          | 是否展示 | 示例                                      |
|--------------|---------|-------------------------------------------|
| 用户意图动作   | ✓ 展示  | 搜索 XXX、生成报告、分析数据、翻译文本       |
| 系统级工具     | ✗ 隐藏  | read_file（读内部配置）、write_file（写临时文件） |
| 技术运维操作   | ✗ 隐藏  | prompt 组装、上下文管理、记忆读写            |
| 框架调度       | ✗ 隐藏  | skill 路由、工具选择、能力匹配              |

### 5.3 聚合规则

连续多次相似的 Agent Loop 动作可以被合并为一个语义步骤卡片：

```
  后端实际执行:                     前端展示:
  ┌──────────────────┐
  │ search("karpathy │
  │   2026 blog")    │
  ├──────────────────┤             ┌──────────────────────────┐
  │ search("karpathy │ ──合并──→  │ ● 搜索 Karpathy 最新观点  →│
  │   2026 talk")    │             └──────────────────────────┘
  ├──────────────────┤
  │ search("karpathy │
  │   AI agent")     │
  └──────────────────┘
```

聚合条件：

| 条件                 | 说明                                      |
|---------------------|-------------------------------------------|
| 连续同类工具          | 连续调用同一工具（如多次 search）              |
| 语义相近              | 多次调用服务于同一用户意图                     |
| 时间窗口              | 连续调用间隔 < 阈值（如 5s）                  |

聚合后的卡片标题由后端生成，使用语义化描述而非工具名。

### 5.4 过滤流程

```
  Agent 执行工具调用
        │
        ▼
  ┌─────────────┐     是
  │ 系统级/框架级？├──────→  丢弃，不下发
  └──────┬──────┘
         │ 否
         ▼
  ┌──────────────┐    是
  │ 与前一步同类？ ├──────→  聚合到前一步骤卡片
  └──────┬───────┘
         │ 否
         ▼
  生成新的 step_card 消息下发前端
```

## 6. 数据流 & 消息类型

### 6.1 消息类型定义

非任务场景下，前后端通信**仅使用 5 种消息类型**：

| 类型              | 方向       | 说明                               |
|------------------|-----------|-------------------------------------|
| `user_message`   | 前端→后端  | 用户发送的消息                       |
| `thinking`       | 后端→前端  | Agent 推理过程（折叠区内容）          |
| `plan_checklist` | 后端→前端  | 计划清单（仅 Plan Mode），含步骤列表  |
| `step_card`      | 后端→前端  | 步骤卡片，含标题、状态、输入、输出     |
| `ai_text`        | 后端→前端  | 最终文本输出（Summary Output）        |

### 6.2 消息体结构

```
user_message:
{
  "type": "user_message",
  "content": "搜索 Karpathy 最新观点并写一篇摘要"
}

thinking:
{
  "type": "thinking",
  "content": "用户想了解 Karpathy 的最新观点，我需要先搜索..."
}

plan_checklist:
{
  "type": "plan_checklist",
  "steps": [
    {"index": 1, "title": "搜索 Karpathy 最新观点", "status": "pending"},
    {"index": 2, "title": "整理要点并归类",       "status": "pending"},
    {"index": 3, "title": "生成摘要报告",          "status": "pending"}
  ]
}

step_card:
{
  "type": "step_card",
  "step_id": "step_001",
  "title": "搜索 Karpathy 2026 最新观点",
  "status": "completed",           // running | completed | failed
  "duration": 3.2,
  "input": { "query": "Karpathy 2026", "max_results": 5 },
  "output": "搜索结果: 1. Karpathy 在..."
}

ai_text:
{
  "type": "ai_text",
  "content": "根据搜索结果，Karpathy 最近在..."
}
```

### 6.3 SSE 数据流（普通模式）

```
  前端                          后端
   │                             │
   │  POST /chat                 │
   │  { user_message }           │
   │ ──────────────────────────→ │
   │                             │
   │  SSE: thinking              │  ← 推理过程（流式）
   │ ←────────────────────────── │
   │                             │
   │  SSE: step_card (running)   │  ← 步骤开始
   │ ←────────────────────────── │
   │                             │
   │  SSE: step_card (completed) │  ← 步骤完成（含 I/O）
   │ ←────────────────────────── │
   │                             │
   │  SSE: ai_text (streaming)   │  ← 最终输出（流式）
   │ ←────────────────────────── │
   │                             │
   │  SSE: [DONE]                │  ← 流结束
   │ ←────────────────────────── │
```

### 6.4 SSE 数据流（计划模式）

```
  前端                          后端
   │                             │
   │  POST /chat                 │
   │  { user_message }           │
   │ ──────────────────────────→ │
   │                             │
   │  SSE: thinking              │  ← 推理过程
   │ ←────────────────────────── │
   │                             │
   │  SSE: plan_checklist        │  ← 计划清单（全部 pending）
   │ ←────────────────────────── │
   │                             │
   │  SSE: step_card (running)   │  ← 步骤 1 开始
   │ ←────────────────────────── │
   │  SSE: plan_checklist        │  ← 清单更新：步骤 1 running
   │ ←────────────────────────── │
   │                             │
   │  SSE: step_card (completed) │  ← 步骤 1 完成
   │ ←────────────────────────── │
   │  SSE: plan_checklist        │  ← 清单更新：步骤 1 ☑
   │ ←────────────────────────── │
   │                             │
   │  SSE: step_card (running)   │  ← 步骤 2 开始
   │ ←────────────────────────── │
   │  ...重复直到全部步骤完成...    │
   │                             │
   │  SSE: ai_text (streaming)   │  ← 最终总结
   │ ←────────────────────────── │
   │                             │
   │  SSE: [DONE]                │
   │ ←────────────────────────── │
```

### 6.5 右侧面板交互流程（数据角度）

```
  用户点击 step_card 的 →
        │
        ▼
  前端从该 step_card 消息中
  提取 input / output
        │
        ▼
  渲染右侧面板（样式 A）
  ┌──────────────────────┐
  │ title + status       │
  │ input (JSON 渲染)    │
  │ output (Markdown)    │
  └──────────────────────┘
        │
  如果 status == "running"
        │
        ▼
  持续监听该 step_id 的
  SSE 更新，实时刷新面板
```

> 右侧面板数据完全来自 `step_card` 消息体，无需额外 API 请求。
