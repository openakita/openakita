# UI 卡片交互调整规范 v2

> 基于 `best-practice-flow-spec-20260310.md` 原有设计，结合 `card_adjust_20260310.md` 中的调整需求，整合为本规范。

---

## 1. 整体布局（不变）

```
┌─────────────┬──────────────────────────────────┬─────────────────────────┐
│  左侧边栏    │         中间聊天区域               │      右侧结果看板        │
│             │                                  │                         │
│ [最佳实践入口]│  对话消息流 + 各类卡片 + 用户输入    │  卡片结果详情面板         │
│ [会话列表]   │                                  │  （点击中间卡片时打开）    │
└─────────────┴──────────────────────────────────┴─────────────────────────┘
```

**重点**: 由中间区域的卡片点击触发展示，卡片类型见如下。

---

## 2. 中间聊天区域 — 卡片类型定义

中间区域包含 4 种卡片类型。所有卡片均为**条状（strip）样式**，嵌入对话消息流中。稍微文字区域加个阴影，右边有个指向符号（→），高度不要太高。其他都是以文字形式展示即可。

### 2.1 普通步骤卡片（StepCard）

**触发场景**: MasterAgent 或 WorkAgent 在 Agent Loop 过程中执行 tool_call 时产生。等同于单 Agent 模式下的步骤卡片。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │   ← 可折叠的思考过程（后续迭代）
│  搜索卡帕西 2026                          [→] │   ← 条状卡片，点击 → 打开右侧面板
│  写PDF文件                                [→] │   ← 条状卡片，点击 → 打开右侧面板
│  一段总结输出巴拉巴拉...                        │   ← LLM 文本输出
└──────────────────────────────────────────────┘
```

**交互行为**:
- 点击某个条状卡片 → 右侧打开**样式 A 面板**，展示该步骤的输入和输出
- Thinking 展开/折叠功能**暂不实现**，后续迭代加入

---

### 2.2 任务卡片（TaskCard）

**触发场景**（3 种，交互行为各不相同）:

| 触发方式 | 说明 | 行为 |
|---------|------|------|
| A. 用户点击左侧最佳实践入口 | 用户主动选择，意图明确 | "引导用户输入任务步骤的要求" ，输入数据后，与用户确认是否执行|
| B. 用户消息命中本会话已有任务 | 如"帮我看看那个API任务"、提到已有任务相关内容 | **唤醒提示**，引导用户打开右侧面板 |
| C. 用户消息命中新的最佳实践 | 当前会话无此任务，系统推荐 | **需确认**，显示 [自由探索] / [使用最佳实践], 如选择[自由探索]则前端携带 `force_route=agent_loop` 重新提交原消息，跳过路由直接进入 `agent_loop` 处理，如选择[使用最佳实践]，则引导输入，输入后与用户确认是否执行 |

#### 触发方式 A — 用户点击最佳实践入口

用户意图明确，任务卡片仅作为信息展示，随后直接进入第 1 步输入收集。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  好的，开始为您执行「API设计流水线」任务           │   ← **LLM 文本提示**
│  ┌────────────────────────────────────────┐  │
│  │  xxxx 任务名                           │  │   ← 任务卡片主体（纯展示）
│  │  任务简要描述 balabala                  │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌─ 步骤输入表单 ─────────────────────────┐  │   ← 紧随其后，直接展示,简洁布局
│  │  开始第1步「需求分析」，请提供以下信息：   │  │
│  │  字段1: [___________]                  │  │
│  │  字段2: [___________]                  │  │
│  │                          [确认提交]     │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

**交互流程**: 点击最佳实践 → 提示 + 任务卡片渲染 + 引导输入 → 用户填写提交 → 再提示用户是否执行 → 用户确认后开始执行，跳到2.4
作为用户的输入消息展示在右侧。

#### 触发方式 B — 唤醒已有任务（提示打开）

用户消息关联到当前会话中已存在的任务，展示该任务卡片，引导用户点击打开右侧面板查看任务详情。**不自动执行任何操作**。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  您提到的内容与当前任务「API设计流水线」相关        │   ← LLM 文本提示
│  ┌────────────────────────────────────────┐  │
│  │  API设计流水线                     [→]  │  │   ← 任务卡片，可点击
│  │  已完成 2/3 步骤                       │  │   ← 显示当前进度/状态
│  └────────────────────────────────────────┘  │
│  一段 LLM 的回复巴拉巴拉...                     │   ← 对用户问题的文字回复
└──────────────────────────────────────────────┘
```

**交互行为**:
- 点击 [→] → 右侧打开**样式 B 面板**，展示该任务的步骤导航和各步骤输入/输出
- 卡片仅起提醒/快捷入口作用，不触发任何执行

#### 触发方式 C — 系统推荐新任务（需确认）

系统自动检测到会话内容匹配某个最佳实践（当前会话中尚无此任务），需要用户决定是否使用。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  检测到你当前的会话命中 xxx 最佳实践              │   ← LLM 文本提示
│  ┌────────────────────────────────────────┐  │
│  │  xxxx 任务名                           │  │   ← 任务卡片主体
│  │  任务简要描述 balabala                  │  │
│  │            [自由探索]    [使用最佳实践]   │  │   ← 两个操作按钮
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

**交互流程**:
- 用户点击 **[使用最佳实践]** → 同触发方式 A 后续流程
- 用户点击 **[自由探索]** → 前端携带 `force_route=agent_loop` 重新提交原消息，跳过路由进入自由对话

**特别注意**：
- 若上下文中已有相关信息 → 预填表单，让用户确认/修改
- 若无相关信息 → 展示空白表单让用户填写

---

### 2.3 任务步骤卡片（TaskStepCard）

**触发场景**: 任务执行过程中，某个步骤（SubAgent）被触发时。也用于用户在对话中要求修改某步骤输出时, router会路由到这各分支。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │   ← 可折叠（后续迭代）
│  任务xxx — 步骤1：xxxx步骤名              [→] │   ← 条状卡片，点击 → 打开右侧面板
│  一段根据用户问题 balabala 的回复                │   ← LLM 文本输出
└──────────────────────────────────────────────┘
```

**交互行为**:
- 点击条状卡片 → 右侧打开**样式 B 面板**，展示任务步骤导航 + 该步骤的输入/输出
---

### 2.4 任务步骤子卡片（任务在执行中）

**触发场景**: 任务步骤（SubAgent）执行态了，在 Agent Loop 中执行的每个 tool_call。跟在任务步骤卡片后面，缩进展示， 执行过程中流式输出。

**视觉样式**:
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │   ← 可折叠（后续迭代）
│  任务xxx — 步骤1：xxxx步骤名              [→] │   ← 父级步骤卡片
│    ├─ 子卡片1：xxxx 子步骤名1             [→] │   ← 缩进，点击 → 右侧面板
│    └─ 子卡片2：xxxx 子步骤名2             [→] │   ← 缩进，点击 → 右侧面板
└──────────────────────────────────────────────┘
```

**交互行为**:
- 点击子卡片 → 右侧打开**样式 C 面板**，展示父步骤标题 + 该子步骤的输入/输出
---


## 3. 右侧结果看板 — 面板样式定义

右侧面板由中间区域卡片的点击触发，根据点击的卡片类型展示不同样式。

### 3.1 样式 A — 普通步骤详情面板

**触发**: 点击中间区域的**普通步骤卡片**（2.1），对应 `agent_loop` 路由

**布局**:
```
┌─────────────────────────────────────────┐
│  任务名|步骤名称                          [✕]  │
│─────────────────────────────────────────│
│                                         │
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  tool_call 的参数 / 输入内容        │ │
│  └─────────────────────────────────────┘ │
│                                         │
│  ┌─ 输出 ─────────────────────────────┐ │
│  │  tool_call 的返回结果 / 输出内容    │ │
│  └─────────────────────────────────────┘ │
│                                         │
└─────────────────────────────────────────┘
```

**特性**: 只读展示，无编辑功能。

---

### 3.2 样式 B — 任务步骤详情面板

**触发**: 点击中间区域的**任务步骤卡片**（2.3）或**唤醒任务卡片**（2.2-B），对应 `recall_task` / `step_interaction` 路由

**布局**:
```
┌─────────────────────────────────────────┐
│  任务名称                          [✕]  │
│─────────────────────────────────────────│
│                                         │
│  步骤导航（水平排列）:                    │
│  [●步骤1 名称] [○步骤2 名称] [○步骤3 名称]│
│    ↑ 当前选中                            │
│─────────────────────────────────────────│
│                                         │
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  该步骤的输入内容                    │ │
│  └─────────────────────────────────────┘ │
│                                         │
│  ┌─ 输出（可编辑）────────────────────┐ │
│  │  该步骤的输出内容                    │ │
│  │                                     │ │
│  │  文本类型 → 直接在面板内编辑         │ │
│  │  数组类型 → 提供 [新增] [删除] 按钮  │ │
│  │                                     │ │
│  │                         [确认更新]   │ │
│  └─────────────────────────────────────┘ │
│                                         │
└─────────────────────────────────────────┘
```

**特性**:
- 步骤导航可点击切换查看不同步骤
- 输出区域可编辑：
  - 文本输出 → 内联文本编辑器
  - 数组输出 → 列表编辑器（支持新增/删除条目）
- 点击 **[确认更新]** → 提交修改到后端

---

### 3.3 样式 C — 任务步骤子卡片详情面板

**触发**: 点击中间区域的**任务步骤子卡片**（2.4）

**布局**:
```
┌─────────────────────────────────────────┐
│  任务xxx — 步骤1：xxxx步骤名       [✕]  │  ← 父步骤标题
│─────────────────────────────────────────│
│                                         │
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  子步骤的 tool_call 参数            │ │
│  └─────────────────────────────────────┘ │
│                                         │
│  ┌─ 输出 ─────────────────────────────┐ │
│  │  子步骤的 tool_call 返回结果        │ │
│  └─────────────────────────────────────┘ │
│                                         │
└─────────────────────────────────────────┘
```

**特性**: 只读展示，顶部标明所属的父步骤。
