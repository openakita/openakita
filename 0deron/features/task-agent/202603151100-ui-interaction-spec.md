# UI 与交互提炼规范（卡片体系）

> 本文仅保留界面结构、卡片样式与交互行为，不涉及实现角色、调度逻辑或内部架构术语。

---

## 1. 页面整体布局

```
┌─────────────┬──────────────────────────────────┬─────────────────────────┐
│  左侧边栏    │         中间聊天区域               │      右侧结果看板        │
│             │                                  │                         │
│ [最佳实践入口]│  对话消息流 + 各类卡片 + 用户输入    │  卡片详情面板             │
│ [会话列表]   │                                  │  （点击中间卡片后打开）   │
└─────────────┴──────────────────────────────────┴─────────────────────────┘
```

- 中间区域负责展示消息与卡片流。
- 右侧看板由“点击中间卡片”触发打开。
- 所有卡片使用条状风格：文字为主、轻阴影、右侧箭头（→）、整体高度较低。

---

## 2. 中间聊天区域卡片类型

中间区域包含 4 类卡片，均嵌入在对话消息流中。

### 2.1 普通步骤卡片（StepCard）

**使用场景**：在常规执行过程中产生的步骤记录。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │
│  搜索资料                                [→] │
│  生成文件                                [→] │
│  一段总结输出文本...                          │
└──────────────────────────────────────────────┘
```

**交互行为**：
- 点击条状步骤卡片（[→]）→ 右侧打开“样式 A：普通步骤详情”。
- Thinking 折叠能力先保留占位，本阶段不实现实际展开/收起。

---

### 2.2 任务卡片（TaskCard）

任务卡片有 3 种触发方式，不同触发方式对应不同交互。

#### A. 用户主动点击“最佳实践入口”

**场景**：用户意图明确，进入任务引导。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  好的，开始为您执行「某任务」                   │
│  ┌────────────────────────────────────────┐  │
│  │  任务名称                               │  │
│  │  任务简要描述                            │  │
│  └────────────────────────────────────────┘  │
│  ┌─ 步骤输入表单 ─────────────────────────┐  │
│  │  开始第1步，请提供以下信息：             │  │
│  │  字段1: [___________]                  │  │
│  │  字段2: [___________]                  │  │
│  │                          [确认提交]     │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

**交互流程**：
- 进入任务后先展示提示文本 + 任务卡片 + 第一步输入表单。
- 用户提交表单后，继续提示“是否执行”。
- 用户确认后开始任务步骤展示。

#### B. 命中当前会话中的已有任务

**场景**：用户消息与本会话已存在任务相关，系统提示可快速查看。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  您提到的内容与当前任务「某任务」相关            │
│  ┌────────────────────────────────────────┐  │
│  │  某任务名称                        [→] │  │
│  │  已完成 2/3 步骤                       │  │
│  └────────────────────────────────────────┘  │
│  一段针对用户问题的文字回复...                 │
└──────────────────────────────────────────────┘
```

**交互行为**：
- 点击 [→] → 右侧打开“样式 B：任务步骤详情”。
- 仅作为提醒与快捷入口，不自动触发执行。

#### C. 推荐新任务（需要用户选择）

**场景**：会话内容匹配到新任务模式，当前会话中尚未创建该任务。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  检测到当前会话匹配某最佳实践                 │
│  ┌────────────────────────────────────────┐  │
│  │  任务名称                               │  │
│  │  任务简要描述                            │  │
│  │            [自由探索]    [使用最佳实践]   │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

**交互流程**：
- 点击 **[使用最佳实践]**：进入 A 的输入与确认流程。
- 点击 **[自由探索]**：回到普通对话处理，不进入任务流程。
- 若上下文已有可复用信息，输入表单应预填；否则显示空表单。

---

### 2.3 任务步骤卡片（TaskStepCard）

**使用场景**：任务执行过程中，某个步骤被触发；或用户要求调整某个步骤结果时。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │
│  任务X — 步骤1：步骤名称                 [→] │
│  一段针对该步骤的说明文本...                  │
└──────────────────────────────────────────────┘
```

**交互行为**：
- 点击条状步骤卡片（[→]）→ 右侧打开“样式 B：任务步骤详情”。

---

### 2.4 任务步骤子卡片（执行中明细）

**使用场景**：某个任务步骤执行时，产生更细的子步骤记录；在父步骤下缩进显示。

**示意图**：
```
┌──────────────────────────────────────────────┐
│  ...Thinking...                          [↓] │
│  任务X — 步骤1：步骤名称                 [→] │
│    ├─ 子卡片1：子步骤名称1              [→] │
│    └─ 子卡片2：子步骤名称2              [→] │
└──────────────────────────────────────────────┘
```

**交互行为**：
- 点击子卡片（[→]）→ 右侧打开“样式 C：子步骤详情”。

---

## 3. 右侧结果看板样式

右侧面板由中间卡片点击触发，根据卡片类型展示 A / B / C 三种样式。

### 3.1 样式 A：普通步骤详情（只读）

**对应卡片**：普通步骤卡片。

```
┌─────────────────────────────────────────┐
│  任务名 | 步骤名                     [✕] │
│─────────────────────────────────────────│
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  本步骤输入内容                     │ │
│  └─────────────────────────────────────┘ │
│  ┌─ 输出 ─────────────────────────────┐ │
│  │  本步骤输出内容                     │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 仅展示输入/输出，不支持编辑。

### 3.2 样式 B：任务步骤详情（可编辑）

**对应卡片**：任务卡片（唤醒已有任务）与任务步骤卡片。

```
┌─────────────────────────────────────────┐
│  任务名称                          [✕]  │
│─────────────────────────────────────────│
│  步骤导航（水平）:                      │
│  [●步骤1] [○步骤2] [○步骤3]            │
│─────────────────────────────────────────│
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  当前步骤输入                        │ │
│  └─────────────────────────────────────┘ │
│  ┌─ 输出（可编辑）────────────────────┐ │
│  │  当前步骤输出                        │ │
│  │                         [确认更新]   │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 步骤导航可点击切换查看不同步骤。
- 输出区支持编辑：
  - 文本：直接编辑。
  - 列表：支持新增/删除条目。
- 点击 **[确认更新]** 提交修改。

### 3.3 样式 C：子步骤详情（只读）

**对应卡片**：任务步骤子卡片。

```
┌─────────────────────────────────────────┐
│  任务X — 步骤1：步骤名称           [✕]  │
│─────────────────────────────────────────│
│  ┌─ 输入 ─────────────────────────────┐ │
│  │  子步骤输入内容                     │ │
│  └─────────────────────────────────────┘ │
│  ┌─ 输出 ─────────────────────────────┐ │
│  │  子步骤输出内容                     │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 顶部显示所属父步骤信息。
- 仅展示，不支持编辑。

---

## 4. 统一交互约束

- 右侧看板始终由“点击中间卡片”触发，不自动抢焦点。
- 条状卡片统一保持低高度、文本优先、右箭头入口一致。
- 可编辑能力仅在“样式 B”中开放，其余面板均只读。
- 任务推荐场景必须先让用户选择，再进入后续流程。
