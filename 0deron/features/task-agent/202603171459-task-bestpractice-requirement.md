当前需求如下：
我想做一个做一个能管理任务执行的多Agent智能体，其大体流程如下：
1. 系统里有根据业务整理的最佳实践（即根据业务沉淀下来的任务模板），由多个子任务组成，每个子任务加载自己的技能和工具，专注于完成自己的事情交互结果
2. 最佳实践的唤醒可以有以下几种方式：
   - 系统指令触发，如“执行任务X”
   - 事件触发，如“当事件Y发生时执行任务X”
   - 根据用户上下文判定触发，如“当用户上下文满足条件Z时执行任务X”
   - 定时触发，如“每天凌晨1点执行任务X” 
   - 用户点击页面上的某个最佳实践触发

3. 每个子任务输出说明
   每个子任务subagent在正常运行完所有步骤并输出完后总结后，另外还一步才能算是完成，即：
   - 简要的说明子任务运行的结果
   - 根据该子任务的每个步骤的输出，结合下游子任务的输入要求，给出json字符串

4. 关于子任务subagent的运行输入要求
   - 每个子任务有自己的输入schema要求，在配置里定义
   - 子任务会接收到一个上游传过来的json格式的字符串，可能是：1）上游子任务运行后传过来的json字符串；2）用户在对话中判断触发任务时，根据用户上下文判断触发，从用户上下文里提取的json字符串。（json的生产是根据接下来要执行的子任务的输入要求来进行整理的）

5. 交互的说明
- 子任务输入：
    每个子任务subagent在运行过程中，会根据自己的输入要求，与用户进行交互。如果用户提供的信息不足，则需要提示用户提供相关信息（ask_user）。
- 子任务输出：
    子任务运行结果后面会有两个按钮（在右侧展示），一个是查看结果，一个是进入下一步子任务subagent的运行。
- [查看结果]按钮：点击后在右侧看板上渲染这个json字符串，用户可以对其进行查看和编辑。这里有个特殊的说明如下：
```
对于这种情况，右侧的网页渲染应该是上面是这个大任务的多个子任务（相对大任务来讲是步骤）的横向引导
下面是当前子任务的json字符串渲染结果（可编辑和保存），大概如下的样式：
┌─────────────────────────────────────────┐
│  任务名称                          [✕]  │
│─────────────────────────────────────────│
│                                         │
│  步骤导航（水平排列）:                    │
│  [●步骤1 名称] [○步骤2 名称] [○步骤3 名称]│
│    ↑ 当前选中                            │
│─────────────────────────────────────────│
│  ┌─ 输出（可编辑）────────────────────┐ │
│  │  该步骤的输出内容                    │ │
│  │                                     │ │
│  │  文本类型 → 直接在面板内编辑         │ │
│  │  数组类型 → 提供 [新增] [删除] 按钮  │ │
│  │                                     │ │
│  │                         [确认更新]   │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```
- [进入下一步]按钮：点击后会直接进入下一个子任务subagent的运行，将当前子任务的json字符串作为输入传递给下一个子任务subagent。(如果在自动模式下，不用用户点击[进入下一步]按钮，直接进入下一个子任务subagent的运行)

6. 注意点：
有普通运行模式和自动模式两种运行方式，界面上可以设置这样的。
- 普通运行模式下，每个子任务subagent在运行完后，会等待用户点击[进入下一步]按钮，才会进入下一个子任务subagent的运行。
- 自动模式下，每个子任务subagent在运行完后，会直接进入下一个子任务subagent的运行，而不用等待用户点击[进入下一步]按钮。
