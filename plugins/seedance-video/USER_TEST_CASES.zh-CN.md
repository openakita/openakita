# 即梦工作室（seedance-video）用户测试用例

> 适用对象：手工测试人员 / 产品验收人员
> 重点约束：全程避开人脸、真人肖像、面部特写、明星、儿童、证件照等内容。推荐使用风景、动物、植物、建筑、机械、抽象图形、食物、玩具、虚拟物体。

## 0. 测试前准备

### 0.1 环境准备

1. 启动 OpenAkita 安装版。
2. 进入「插件」或侧边栏中的「即梦工作室」。
3. 打开「设置」页。
4. 确认顶部没有明显的插件加载错误。

预期效果：

- 页面可正常显示「创建 / 分镜工作台 / 任务列表 / 素材库 / 提示词指南 / 设置」等标签页。
- 如果未配置 Ark API Key，会看到黄色提示「请先配置 API Key」。

说明：

- API Key 是火山引擎 Ark 的密钥，用于真实调用视频生成接口；涉及实际费用时，只跑少量短视频用例。
- FFmpeg 是视频处理工具，主要用于分镜长视频拼接和本地预览辅助，不影响普通文生视频创建。

### 0.2 测试素材准备

本测试需要 4 个本地文件。推荐**先用插件自己生成素材**，这样测试流程更接近真实用户，也不依赖外部素材下载。

最终需要准备的文件名：

1. `forest_start.png`：森林小路首帧图。
2. `forest_end.png`：同一森林小路的傍晚尾帧图。
3. `object_ref.png`：一个非人脸物体参考图。
4. `source_landscape.mp4`：一段 3-5 秒非人脸源视频。

文件要求：

- 图片建议小于 5 MB。
- 视频建议小于 20 MB，便于上传和测试。
- 不使用真人、人脸、肖像、面部局部、动漫人脸、拟真人物。
- 文件名用英文，避免路径编码问题。

### 0.2.1 插件真实模式名称对照

测试时请以页面按钮文字为准。当前「创建」页的生成模式按钮依次是：

| UI 按钮 | 测试含义 | 需要的素材 |
| --- | --- | --- |
| 文生视频 | 只用文字生成视频 | 不需要上传素材 |
| 首帧 | 上传一张首帧图片，让视频从这张图开始运动 | `forest_start.png` |
| 首尾帧 | 上传首帧和尾帧，让 AI 补全中间过渡 | `forest_start.png` + `forest_end.png` |
| 多模态 | 上传图片或视频作为参考，再结合提示词生成 | `object_ref.png` 或视频 |
| 编辑 | 上传源视频，对已有视频做风格/内容编辑 | `source_landscape.mp4` |
| 延长 | 上传源视频，向后延续原视频内容 | `source_landscape.mp4` |

注意：

- 文档后续步骤统一使用上面的 UI 名称，不使用接口层叫法。
- 如果页面显示文案发生变化，以实际按钮为准，但测试目的不变。

### 0.2.2 推荐素材生成路线

优先按下面路线准备素材：

1. 用 `tongyi-image` 插件生成 `forest_start.png`、`forest_end.png`、`object_ref.png`。
2. 用本插件「即梦工作室」的「文生视频」生成 `source_landscape.mp4`。
3. 如果任一插件不可用，再使用免版权素材网站下载替代素材。

这条路线的好处：

- 可以顺便验证 `tongyi-image` 到本插件的跨插件素材工作流。
- 不需要搜索真人或人像素材，天然规避人脸内容。
- 视频素材可以由本插件自己生成，再作为「编辑 / 延长」的输入，形成一步套一步的完整测试链路。

### 0.2.3 用 tongyi-image 生成图片素材

进入 `tongyi-image` 插件，分别生成以下 3 张图片。生成后下载到本地测试目录，例如：

```text
D:\OpenAkita-test-assets\seedance-video\
```

#### 图片 1：forest_start.png

用途：「首帧」模式上传图、「首尾帧」模式首帧图。

在 `tongyi-image` 输入：

```text
清晨的森林小路，石板路通向远处，柔和晨雾，树叶带着露水，阳光从树枝间洒下，写实动画风格，16:9 横图，高细节，无人，无人脸，无人物，无肖像
```

生成后保存为：

```text
forest_start.png
```

验收标准：

- 画面主体是森林小路。
- 没有人、脸、雕像脸、海报脸、动物拟人脸。
- 横图优先，16:9 更适合视频测试。

#### 图片 2：forest_end.png

用途：「首尾帧」模式尾帧图。

在 `tongyi-image` 输入：

```text
傍晚的森林小路，与清晨森林小路保持相似构图，金色夕阳穿过树枝，石板路延伸到远处，暖色调，写实动画风格，16:9 横图，高细节，无人，无人脸，无人物，无肖像
```

生成后保存为：

```text
forest_end.png
```

验收标准：

- 与 `forest_start.png` 主题相同，光照从清晨变为傍晚。
- 没有人脸、人物、肖像。
- 用于测试首尾帧过渡是否自然。

#### 图片 3：object_ref.png

用途：多模态图片参考。

在 `tongyi-image` 输入：

```text
一个蓝色陶瓷杯放在木桌中央，旁边有少量绿色叶子装饰，柔和棚拍灯光，干净背景，产品摄影风格，16:9 横图，高细节，无人，无人脸，无人物，无手部
```

生成后保存为：

```text
object_ref.png
```

验收标准：

- 主体是蓝色陶瓷杯或类似静物。
- 不出现手、人、脸、身体局部。
- 背景简单，便于多模态测试观察主体是否被引用。

可选替代物体提示词：

```text
一辆红色玩具小车放在白色桌面上，柔和摄影灯光，干净背景，产品摄影风格，16:9 横图，高细节，无人，无人脸，无人物，无手部
```

```text
一只纸船放在浅蓝色水面上，周围有少量漂浮叶片，清新明亮，写实插画风格，16:9 横图，高细节，无人，无人脸，无人物
```

### 0.2.4 用本插件生成视频素材 source_landscape.mp4

`source_landscape.mp4` 建议直接用本插件「文生视频」生成，再下载为本地视频。这样后续「编辑」和「延长」就是完整的链式测试。

步骤：

1. 进入「即梦工作室」。
2. 点击「创建」。
3. 选择「文生视频」。
4. 模型使用默认模型。
5. 比例选择 `16:9`。
6. 时长选择 `5 秒`。
7. 分辨率选择 `720p`。
8. 数量选择 `1`。
9. 输入下面提示词。
10. 点击创建任务。
11. 到「任务列表」等待任务完成。
12. 打开任务详情，下载视频。
13. 将下载文件重命名为 `source_landscape.mp4`。

输入提示词：

```text
无人山谷中的小溪缓慢流动，水面反射天空和树叶，远处有柔和晨雾，镜头沿着溪流平稳向前推进，写实动画风格，画面自然安静，16:9，不出现人类，不出现人脸，不出现人物，不出现肖像
```

预期效果：

- 生成一段 3-5 秒风景视频。
- 画面没有人、脸、肖像、人体局部。
- 下载后可作为「编辑」和「延长」用例的源视频。

备用视频提示词：

```text
无人的海边灯塔，海浪缓慢拍打礁石，天空云层轻轻移动，镜头缓慢推近灯塔，电影感，写实动画风格，16:9，不出现人类，不出现人脸，不出现人物，不出现肖像
```

```text
无人桌面上的红色玩具小车，柔和灯光，镜头缓慢环绕玩具小车，背景干净，产品展示视频风格，16:9，不出现人类，不出现人脸，不出现手部，不出现人物
```

### 0.2.5 外部免版权素材兜底

如果 `tongyi-image` 或本插件暂时不可用，可以从免版权素材网站下载替代素材。

推荐网站：

- [Pexels](https://www.pexels.com/)

- [Unsplash](https://unsplash.com/)（主要是图片）
- [Videvo](https://www.videvo.net/)（主要是视频）

搜索关键词：

```text
forest path no people
empty forest trail
mountain stream no people
ocean waves no people
ceramic cup product photo
toy car product photo
paper boat water
```

下载前检查：

- 不要选有人、背影、面部、雕像脸、广告牌人像、照片墙的素材。
- 视频里不要出现路人、游客、主播、电视屏幕里的人脸。
- 下载后统一重命名为本文档要求的 4 个文件名。

## 1. 设置页测试

### TC-SET-001 保存 Ark API Key

目的：验证 API Key 能真实保存并回显，不再只显示“设置已保存”但实际未落库。

步骤：

1. 点击「设置」标签页。
2. 在「ARK API Key」输入框粘贴有效 Ark API Key。
3. 点击「保存」。
4. 等待右上角提示。
5. 切换到「创建」页，再切回「设置」页。
6. 点击「显示」查看输入框内容是否仍存在。

输入：

```text
使用测试环境有效 Ark API Key
```

预期效果：

- 成功时提示「设置已保存」。
- 页面右上角不再显示「请先配置 API Key」。
- 切换页面后 Key 仍存在，显示为保存后的值。
- 如果保存失败，toast 应明确显示 HTTP 状态、网络错误或“回读不一致”，不能再假成功。

### TC-SET-002 保存空白 Ark API Key

目的：验证用户误操作时提示友好。

步骤：

1. 点击「设置」。
2. 清空「ARK API Key」输入框，或只输入空格。
3. 点击「保存」。

输入：

```text
三个空格
```

预期效果：

- 显示红色错误提示：「API Key 不能为空白」或后端返回的空白 Key 提示。
- 不应显示「设置已保存」。
- 原有已保存 Key 不应被空白覆盖。

### TC-SET-003 修改默认推理模式

目的：验证除 API Key 外的普通配置项也能保存。

步骤：

1. 打开「设置」。
2. 找到「默认推理模式」或「推理模式」下拉框。
3. 从「在线推理」切换为「离线 / Flex」。
4. 等待 toast。
5. 切换到其他标签页，再切回「设置」。
6. 观察下拉框当前选项。
7. 再切回「在线推理」，重复保存一次。

预期效果：

- 每次切换后提示「设置已保存」。
- 切换页面后下拉框保持刚才选择的值。
- 如果保存失败，应提示类似「service_tier_default 服务端回读为 X，与你设置的 Y 不一致」。

### TC-SET-004 修改轮询间隔

目的：验证轮询间隔配置可保存。

步骤：

1. 打开「设置」。
2. 找到「轮询间隔」下拉框。
3. 选择「30 秒」。
4. 切换页面后返回「设置」。

预期效果：

- 显示「设置已保存」。
- 下拉框保持「30 秒」。

### TC-SET-005 修改提示词优化等级

目的：验证提示词优化等级配置可保存。

步骤：

1. 打开「设置」。
2. 找到「提示词优化等级」。
3. 选择「轻量」。
4. 切换页面后返回「设置」。
5. 再选择「专业」恢复默认。

预期效果：

- 每次切换都显示保存成功。
- 页面返回后仍显示刚保存的选项。

### TC-SET-006 FFmpeg 检测、安装、卸载

目的：验证 FFmpeg 系统组件流程，特别是 Windows 安装版下首次安装和卸载。

步骤：

1. 打开「设置」。
2. 滚动到「系统组件」区域。
3. 查看 FFmpeg 当前状态。
4. 如果未安装，点击「一键安装」。
5. 观察安装日志，等待状态结束。
6. 如果显示已安装，点击「卸载」并确认。
7. 观察卸载日志，等待状态结束。
8. 再点击「重新检测」或刷新设置页。

预期效果：

- 安装过程中显示「安装中」和命令日志。
- 安装成功后显示 FFmpeg 版本和路径。
- 卸载成功后显示未安装或可安装状态。
- 不能出现“第一次失败、第二次才成功”的误报；如果失败，应显示明确日志，而不是只显示「已安装」。

备注：

- Windows 安装命令应类似：

```powershell
winget install --id Gyan.FFmpeg -e --accept-source-agreements --accept-package-agreements
```

- Windows 卸载命令应类似：

```powershell
winget uninstall --id Gyan.FFmpeg -e --silent --accept-source-agreements
```

## 2. 创建页测试

以下用例会调用 Ark API，可能产生费用。建议每个模式只生成 1 个短视频，时长选 5 秒，分辨率选 720p。

统一基础参数：

- 模型：Seedance 2.0 或页面默认模型。
- 比例：16:9。
- 时长：5 秒。
- 分辨率：720p。
- 数量：1。
- 水印：关闭或默认。
- 音频：默认即可。
- 不输入任何人脸、真人、肖像相关内容。

### TC-CREATE-001 文生视频

目的：验证最基础的文生视频任务创建。

步骤：

1. 点击「创建」标签页。
2. 选择「文生视频」模式。
3. 在提示词输入框输入下面内容。
4. 保持基础参数。
5. 点击「创建任务」或页面上的生成按钮。
6. 跳转或手动进入「任务列表」查看任务。

输入提示词：

```text
一只橙色小猫在阳光明媚的花园里追逐蝴蝶，镜头缓慢跟随，画面温暖明亮，写实动画风格，不出现人类和人脸
```

预期效果：

- 创建成功，出现任务 ID。
- 任务列表中新增一条「生成中」任务。
- 任务完成后状态变为「完成」，可预览视频。

### TC-CREATE-002 首帧：缺少首帧图片

目的：验证用户未上传图片时提示友好。

步骤：

1. 点击「创建」。
2. 点击生成模式里的「首帧」。
3. 不上传任何图片。
4. 输入提示词并点击创建。

输入提示词：

```text
让画面中的森林小路逐渐出现晨雾，树叶轻轻摇动，镜头缓慢向前推进
```

预期效果：

- 不应创建任务。
- 显示明确错误：需要先上传首帧图片。

### TC-CREATE-003 首帧：正常上传首帧

目的：验证图片上传后可创建「首帧」任务。

步骤：

1. 点击「创建」。
2. 点击生成模式里的「首帧」。
3. 上传 `forest_start.png`。
4. 等待上传区域显示「已就绪 / Ready」。
5. 输入提示词。
6. 点击创建。
7. 到「任务列表」查看任务状态。

输入提示词：

```text
基于这张森林小路图片生成 5 秒视频，晨雾从远处缓慢飘来，树叶轻轻晃动，镜头平稳向前，不出现人类和人脸
```

预期效果：

- 上传后显示文件名和大小。
- 创建成功。
- 素材库中可看到上传的图片素材。
- 任务完成后视频保持森林主题，不出现人脸。

### TC-CREATE-004 首尾帧：缺少尾帧

目的：验证首尾帧模式缺素材时提示明确。

步骤：

1. 点击「创建」。
2. 点击生成模式里的「首尾帧」。
3. 只上传 `forest_start.png`，不上传尾帧。
4. 输入提示词并创建。

输入提示词：

```text
从清晨森林过渡到傍晚森林，光线逐渐变暖，镜头缓慢推进
```

预期效果：

- 不应创建任务。
- 显示明确错误：「首尾帧模式需要上传首帧和尾帧图片」或等价中文提示。

### TC-CREATE-005 首尾帧：正常创建

## 6. 分镜工作台测试

分镜功能会拆成多个视频任务，费用和耗时比单条视频更高。建议先用 2-3 段短故事测试。

### TC-STORY-001 AI 分镜拆解

目的：验证故事能拆解为多段分镜，并且不会生成重复段落。

步骤：

1. 点击「分镜工作台」。
2. 在「故事 / 脚本内容」输入下面文本。
3. 目标总时长选择 10 或 15 秒。
4. 每段时长选择 5 秒。
5. 风格选择默认或「写实动画」。
6. 点击「AI 分镜拆解」。

输入故事：

```text
一只橙色小猫在无人的花园中探索。第一段，它沿着石板路闻一朵蓝色小花；第二段，它追逐一只蝴蝶穿过草地；第三段，它在树荫下趴在木凳旁休息。全片温暖明亮，不出现人类和人脸。
```

预期效果：

- 显示拆解进度。
- 生成约 2-3 段分镜。
- 每段提示词都围绕小猫、花园、蝴蝶、树荫，不出现人脸。
- 如果 LLM 返回重复段落，系统应合并重复并显示提示。

### TC-STORY-002 编辑分镜

步骤：

1. 在分镜结果页展开第一段。
2. 修改提示词中的一个细节，例如把「蓝色小花」改成「黄色小花」。
3. 点击「添加段」新增一段。
4. 输入新的非人脸段落。
5. 点击「上移 / 下移」调整顺序。
6. 删除新增段。

新增段提示词：

```text
小猫停在小池塘边，看见水面倒映着天空和树叶，镜头从低角度缓慢推进，不出现人类和人脸
```

预期效果：

- 编辑后内容保留。
- 添加、移动、删除都生效。
- 分镜数量显示正确。

### TC-STORY-003 并行独立生成

目的：验证分镜并行生成不会重复提交同一批任务。

步骤：

## 9. 非人脸内容检查清单

每个创建类用例提交前都检查一次：

- 没有「人、人物、女孩、男孩、脸、面部、肖像、表情、眼睛特写、嘴唇、皮肤、明星、儿童」等词。
- 图片 / 视频素材中没有真人、动漫人脸、照片墙、人像海报、电视屏幕中的脸。
- 如需角色，使用「小猫、小狗、机器人、玩具车、纸船、陶瓷杯、森林、城市、建筑、风景」。

## 10. 验收标准

本轮测试通过的最低标准：

1. 设置项保存必须真实回读一致；不能只显示成功 toast。
2. API Key、默认推理模式、轮询间隔、提示词优化等级至少各保存成功一次。
3. FFmpeg 安装 / 卸载失败时必须有明确日志；不能出现静默失败。
4. 文生视频、首帧、首尾帧、多模态、编辑、延长至少各覆盖一次缺素材提示和一次正常创建。
5. 任务列表能查看、刷新、筛选、删除。
6. 素材库能显示上传素材，并支持类型筛选。
7. 分镜工作台不会重复提交同一组分镜任务。
8. 所有测试内容均不包含人脸或真人肖像。
