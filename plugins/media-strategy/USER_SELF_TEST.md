# 融媒智策用户自测文档

> 适用插件：`plugins/media-strategy`
> 插件名称：融媒智策 / Media Strategy
> 测试目标：让普通用户按步骤完整跑通“信源准备 -> 热点抓取 -> 热点雷达 -> 信源复核 -> 简报 -> 策研采编 -> 报告历史 -> 设置与存储 -> Agent 工具调用”的全流程。

## 1. 自测前准备

### 1.1 环境准备

1. 启动 OpenAkita 后端服务。
   - 推荐命令：`openakita serve`
   - 作用：启动 FastAPI（Python 后端 Web 框架）服务，插件 UI（前端页面）会通过后端 API（程序接口）读写数据。

2. 确认插件已加载。
   - 打开插件页面，进入“融媒智策”。
   - 页面右上角健康状态应显示类似：`运行中 · v0.1.0 · N/M 源`。
   - 如果显示“插件未就绪”，等待 5-10 秒后刷新；仍失败时到插件管理确认已批准 `routes.register`、`tools.register`、`data.own`、`brain.access` 等权限。

3. 确认网络可访问外部 RSS/HTML 新闻源。
   - RSS 是网站公开订阅格式，用来稳定获取标题、摘要、发布时间和原文链接。
   - 本插件会访问央视、人民网、新华网、RSSHub、BBC、联合早报等公开地址；如果公司网络有代理或拦截，部分源可能失败，属于可记录问题。

4. 可选：确认大模型可用。
   - 在“设置”页的运行信息里看 `Brain 状态`。
   - Brain 是 OpenAkita 内部的大模型调用入口。可用时，简报、复核、策研采编会生成更完整文本；不可用时会降级为规则整理结果，仍应可用。

### 1.2 需要准备的输入内容

建议按下面这组输入跑完整自测：

| 用途 | 输入内容 |
|---|---|
| 热点关键词 | `台海`、`政策`、`关税`、`AI`、`热搜` |
| 复核主题 | `台海最新动态复核` |
| 专题简报主题 | `24小时台海与政策重点` |
| 策研采编主题 | `台海最新动态` |
| 内容形态 | 依次选择 `短视频`、`采访`、`深度文章`、`直播脚本` |
| 语气 | `稳健客观`、`面向本地融媒体编辑，克制不煽动` |
| 自定义套餐名称 | `自测地缘安全` |
| 自定义套餐英文名 | `Geo Test` |
| 自定义套餐描述 | `用于测试跨区域地缘与政策新闻采编链路` |
| 自定义套餐关键词 | `地缘, 冲突, 外交, 制裁, 政策` |
| 自定义数据目录 | Windows 示例：`D:\OpenAkita\.tmp\media-strategy-user-test` |

### 1.3 需要准备的素材

本插件主要处理“新闻源与新闻条目”，不是上传图片/视频文件的插件。因此素材重点是 RSS 地址、主题词和可观察的报告输出。

| 素材 | 推荐值 | 用途 |
|---|---|---|
| 可用 RSS 地址 A | `https://feeds.bbci.co.uk/zhongwen/simp/rss.xml` | 测试添加自定义 RSS 源 |
| 可用 RSS 地址 B | `https://feeds.bbci.co.uk/news/world/rss.xml` | 测试编辑自定义源 URL |
| 应被拒绝的 URL | `http://localhost:8080/rss` | 测试安全校验，应拒绝本机地址 |
| 应被拒绝的 URL | `ftp://example.com/feed.xml` | 测试协议校验，应只允许 http/https |
| 可观察报告 | 简报、信源复核清单、策研采编计划 | 测试报告历史、预览、落盘 |
| 截图素材 | 每个阶段截图 1 张 | 方便回归记录和问题反馈 |

注意：RSSHub 的 `rsshub.app` 有时会限流或不可达。遇到个别源失败时，不要立刻判定插件失败，应看“失败源数量”和具体错误。

## 2. 测试范围总览

本次自测需要覆盖以下功能：

1. 插件健康检查、依赖状态、数据目录状态。
2. 内置套餐启停：台海观察、时政政策、经济财经、国际局势、科技产业、平台热点。
3. 自定义套餐：新建、编辑、克隆内置套餐、删除。
4. 信源管理：同步内置源、添加自定义 RSS、编辑、启停、批量启停、删除自定义源。
5. 抓取：RSS 源抓取和内置 HTML 列表源抓取。
6. 热点雷达：按套餐、时间窗口、关键词查询。
7. 选中热点：默认前三条、手动勾选、清空选择。
8. AI 选题分析：对 Top N 热点簇批量调用主程序大模型，并观察任务进度。
9. 信源复核：基于选中 article_id 生成复核清单。
10. 简报：早报、午报、晚报、专题简报。
11. 策研采编：短视频、采访、深度文章、直播脚本。
12. 报告：历史列表、预览、报告文件输出。
13. 设置：抓取超时、并发、LLM 温度、User-Agent、自定义数据目录、打开目录、存储统计。
14. Agent 工具：在聊天中调用 11 个 `media_strategy_*` 工具。
15. 异常边界：无网络、非法 URL、空结果、Brain 不可用、内置源/内置套餐不可删除。

## 3. UI 自测步骤

### 3.1 健康检查与初始化

操作：

1. 打开“融媒智策”插件页面。
2. 观察顶部标题、口号、图标、健康状态。
3. 切到“设置”页，观察运行信息和依赖状态。

预期结果：

- 顶部显示“融媒智策”和口号“全源融聚・智察热点・策研采编”。
- 健康状态显示 `运行中`，版本为 `v0.1.0`。
- 设置页显示当前数据目录、默认目录、Brain 状态、依赖状态。
- `feedparser`、`bs4` 依赖如果显示 pending，稍后刷新后应变为 ok；如果长期 pending，记录为依赖安装问题。

记录：

- 健康状态文本：
- 已启用源数量：
- 数据目录：
- Brain 状态：

### 3.2 套餐管理

操作 A：启停内置套餐。

1. 进入“设置”页。
2. 在“套餐”区域点击 `经济财经`，使其启用。
3. 点击 `科技产业`，使其启用。
4. 再次点击 `科技产业`，使其停用。

预期结果：

- 被启用的套餐卡片高亮。
- 被停用的套餐卡片取消高亮。
- 页面弹出“已启用/已停用”的提示。
- 回到“雷达”页，套餐下拉中能看到这些套餐。

操作 B：新建自定义套餐。

1. 点击 `+ 新建套餐`。
2. 输入：
   - 套餐名称：`自测地缘安全`
   - 英文名：`Geo Test`
   - 描述：`用于测试跨区域地缘与政策新闻采编链路`
   - 关键词：`地缘, 冲突, 外交, 制裁, 政策`
   - 状态：启用
3. 点击“保存”。

预期结果：

- 出现一个带“自定义”标记的新套餐。
- 套餐可在“添加信源”的套餐选择 chip 中被选中。
- 套餐可在雷达/简报下拉中选择。

操作 C：克隆内置套餐。

1. 在 `台海观察` 卡片上点击 `克隆为我的套餐`。
2. 套餐名称保持默认或改为 `我的台海自测`。
3. 点击“保存”。

预期结果：

- 新增一个自定义套餐。
- 关键词、描述继承自原内置套餐或可编辑。

操作 D：编辑与删除自定义套餐。

1. 点击 `自测地缘安全` 的“编辑”。
2. 将描述改为：`已编辑：地缘、安全、外交信号追踪`。
3. 点击“保存”。
4. 再次打开编辑弹窗，点击“删除”，确认删除。

预期结果：

- 编辑后描述更新。
- 删除后该套餐从列表中消失。
- 删除自定义套餐不会删除信源，只会从信源的套餐标签中剥离该套餐。

边界验证：

- 尝试删除内置套餐：UI 不应提供删除按钮；如果通过接口测试，应返回“内置套餐不可删除”。

### 3.3 信源管理

操作 A：同步内置源。

1. 进入“设置”页。
2. 点击 `同步内置源`。
3. 展开“信源列表”。

预期结果：

- 页面提示“内置信源已同步”。
- 信源列表数量不少于 37 个。
- 能看到 RSS 源和 HTML 源，例如：
  - `央视国内新闻`
  - `新华网 台湾`
  - `人民网 时政`
  - `中国台湾网 即时报道`
  - `东南网 台海频道`
  - `台海网 台湾新闻`
  - `RSSHub 微博热搜`

操作 B：添加自定义 RSS 源。

1. 在“添加 RSS 源”区域输入：
   - 源名称：`自测 BBC 中文`
   - RSS URL：`https://feeds.bbci.co.uk/zhongwen/simp/rss.xml`
   - 套餐：选择 `国际局势`，也可同时选择 `时政政策`
2. 展开“高级设置”。
3. 将权重设置为 `0.70`。
4. 点击 `添加并校验`。

预期结果：

- 页面提示“RSS 源已添加”。
- 信源列表出现 `自测 BBC 中文`，带“自定义”标记。
- 权重显示约 `0.70`。
- URL 完整显示。

操作 C：非法 URL 安全校验。

1. 源名称输入：`非法 localhost`
2. RSS URL 输入：`http://localhost:8080/rss`
3. 任意选择一个套餐，点击 `添加并校验`。

预期结果：

- 添加失败。
- 错误说明应表达“host is not allowed”或“private/reserved IP”。
- 信源列表不应新增该源。

继续测试：

1. 将 RSS URL 改成：`ftp://example.com/feed.xml`
2. 点击 `添加并校验`。

预期结果：

- 添加失败。
- 错误说明应表达只允许公开 `http` 或 `https` URL。

操作 D：编辑自定义信源。

1. 找到 `自测 BBC 中文`，点击“编辑”。
2. 名称改为：`自测 BBC World`。
3. URL 改为：`https://feeds.bbci.co.uk/news/world/rss.xml`。
4. 套餐改为只选 `国际局势`。
5. 展开高级设置，权重改为 `0.78`。
6. 点击“保存”。

预期结果：

- 信源名称、URL、套餐、权重更新。
- 保存后信源列表刷新。

操作 E：启停与批量启停。

1. 找到 `自测 BBC World`，点击“停用”。
2. 再点击“启用”。
3. 在批量操作下拉选择 `国际局势`。
4. 点击“套餐内全部停用”。
5. 再点击“套餐内全部启用”。

预期结果：

- 单个信源启停状态变化。
- 批量操作提示影响了 N 个信源。
- 顶部健康状态中的已启用源数量随之变化。

操作 F：删除自定义信源。

1. 找到 `自测 BBC World`，点击“编辑”。
2. 点击“删除”并确认。

预期结果：

- 自定义信源从列表中消失。
- 内置信源没有删除按钮；不能删除内置信源。

### 3.4 设置与存储

操作 A：修改普通设置。

1. 进入“设置”页。
2. 输入：
   - 抓取超时：`20`
   - 抓取并发：`4`
   - LLM 温度：`0.2`
   - RSS User-Agent：`OpenAkita-MediaStrategy-UserTest/0.1`
3. 点击“保存设置”。
4. 点击“重新加载”。

预期结果：

- 页面提示“设置已保存”。
- 重新加载后设置值仍保留。
- LLM 温度影响简报、复核、采编计划生成时的随机性；0.2 表示偏稳定。

操作 B：自定义数据目录。

1. 点击“浏览”。
2. 进入 `D:\OpenAkita\.tmp`，如果没有则选择一个存在的父目录。
3. 在弹窗底部输入新建文件夹名称：`media-strategy-user-test`。
4. 点击“新建”。
5. 进入新建目录，点击“使用此目录”。
6. 点击“保存设置”。

预期结果：

- 页面提示“设置已保存，数据目录变更需重新加载插件生效”。
- 目录状态显示“等待重新加载”。
- 重新加载插件后，当前数据目录应变成新目录。

操作 C：存储统计与打开目录。

1. 点击“刷新存储统计”。
2. 点击“数据目录”或“报告输出”统计卡。
3. 点击“打开”。

预期结果：

- 统计卡显示大小和文件数。
- 系统文件管理器打开对应目录。
- 生成过报告后，`outputs/YYYY-MM-DD/` 下能看到 Markdown 文件。

边界验证：

1. 自定义数据目录输入相对路径：`tmp/media-strategy`
2. 点击保存。

预期结果：

- 保存失败。
- 提示要求填写绝对路径，例如 `D:\media-strategy`。

### 3.5 抓取与热点雷达

操作 A：抓取台海套餐。

1. 进入“雷达”页。
2. 套餐选择：`台海观察`
3. 时间窗口选择：`24 小时`
4. 关键词留空。
5. 点击 `拉取已启用源`。

预期结果：

- 按钮显示“拉取中...”。
- 完成后“信源复核”区域显示类似：`拉取完成：新增 X，失败源 Y`。
- 指标区显示热点数、已启用源、最近新增、失败源。
- 热点榜出现文章卡片，包含标题、score、risk、source_id、时间和摘要。
- 最近任务出现 `ingest · done`。

如果热点为空：

1. 把时间窗口改为 `72 小时` 或 `7 天`。
2. 点击 `生成热点雷达`。
3. 如果仍为空，换 `全部套餐` 或启用更多套餐后重新拉取。

操作 B：按套餐生成热点雷达。

依次测试以下组合：

| 套餐 | 时间窗口 | 关键词 |
|---|---|---|
| 全部套餐 | 24 小时 | 留空 |
| 台海观察 | 72 小时 | 留空 |
| 时政政策 | 24 小时 | `政策` |
| 经济财经 | 72 小时 | `关税` |
| 科技产业 | 72 小时 | `AI` |
| 平台热点 | 24 小时 | `热搜` |

预期结果：

- 有关键词时走搜索新闻，结果标题/摘要应与关键词相关。
- 无关键词时按时间和套餐展示最新热点。
- 每条热点标题可点击打开原文链接。
- `risk=high` 的内容仅作线索，需要人工复核。

操作 C：选择热点。

1. 观察热点榜默认勾选前三条。
2. 手动取消一条，再勾选另一条。
3. 点击“清空”。
4. 再手动选择 2-3 条热点。

预期结果：

- “已选择 N 条”文本实时更新。
- “策研采编”页也能显示相同选择状态。

### 3.6 AI 选题分析

操作：

1. 在“雷达”页先生成热点雷达。
2. 在“AI 选题分析”卡片中输入：
   - 分析 Top N：`10`
   - 最少来源数：`1`，如果只想看多源交叉印证可填 `2`
   - 每簇证据：`5`
3. 点击 `AI 分析 Top 热点`。

预期结果：

- 页面立即显示任务 ID、百分比和当前阶段。
- 任务阶段应至少出现：
  - `规则聚类中：正在筛选高价值热点`
  - `证据整理中：已选出 N 个热点簇`
  - `大模型分析中：通常需要 30-90 秒，请不要关闭页面`
  - `报告保存中`
- 完成后页面渲染 `AI 选题分析报告`。
- 报告历史新增 `ai_topic_analysis`。
- 如果 Brain 不可用，应生成规则降级报告，`source=fallback`，页面不应卡死。

成本观察：

- 这个功能不是逐条新闻调用大模型，而是先规则聚类后只分析 Top N 热点簇。
- 建议常规测试使用 Top 5-10；压力测试再使用 Top 20。

### 3.7 信源复核

操作：

1. 在“雷达”页选择 2-3 条热点。
2. 复核主题输入：`台海最新动态复核`。
3. 点击 `生成复核清单`。

预期结果：

- 输出一份 Markdown 风格的复核清单。
- 内容应包含：
  - 已有来源、发布时间、链接。
  - 是否多源交叉印证。
  - 缺少哪些证据。
  - 下一步应查什么官方口径、当事方回应、历史背景、数据口径。
  - `可报道 / 需等待 / 仅作线索` 之类建议。
- 最近任务出现 `verify_pack · done`。
- 报告历史新增一条 `verify_pack` 报告。

边界验证：

1. 点击“清空”，不选择热点。
2. 复核主题输入：`政策发布会`
3. 点击 `生成复核清单`。

预期结果：

- 插件会按主题检索相关文章生成复核清单。
- 如果找不到相关文章，报告应为空或只给规则提示，不应页面崩溃。

4. 点击 `先刷新热点`。

预期结果：

- 热点数据刷新，不生成报告。
- 如果刷新失败，页面应弹出明确错误。

操作 B：生成四类简报。

依次输入并生成：

| 简报时段 | 套餐范围 | 时间窗口 | 条数 |
|---|---|---|---|
| 早报 | 全部套餐 | 24 小时 | 20 |
| 午报 | 时政政策 | 24 小时 | 20 |
| 晚报 | 国际局势 | 72 小时 | 20 |
| 专题简报 | 台海观察 | 168 小时 | 30 |

预期结果：

- 每次点击 `生成简报` 后，简报区域显示完整报告。
- 报告应保留原文链接。
- 如果 Brain 可用，报告应有采编判断和分类；如果 Brain 不可用，应显示“规则整理结果”但仍有链接。
- 最近任务出现 `daily_brief · done`。
- 报告历史新增对应报告。

## 4. Agent 聊天工具自测

Agent 工具是指用户在聊天中用自然语言让 OpenAkita 调用插件能力。下面每条都要在聊天窗口测试一次。

### 4.1 列出套餐和信源

输入：

```text
帮我查看融媒智策当前有哪些套餐、信源，各自启用状态如何。
```

预期结果：

- Agent 调用 `media_strategy_list_sources`。
- 回复包含套餐、信源数量、启用状态。
- 不凭记忆猜测信源状态。

### 4.2 订阅和取消套餐

输入：

```text
帮我启用融媒智策的经济财经套餐，并关闭平台热点套餐。
```

预期结果：

- Agent 调用 `media_strategy_subscribe_package` 至少两次，分别处理 `economy=true`、`platform=false`。
- UI 设置页刷新后状态一致。

### 4.3 添加自定义 RSS 源

输入：

```text
帮我把 BBC 中文 RSS 加到融媒智策国际局势套餐里，名称叫“自测 BBC 中文”，地址是 https://feeds.bbci.co.uk/zhongwen/simp/rss.xml
```

预期结果：

- Agent 调用 `media_strategy_add_feed`。
- URL 经过安全校验。
- 新信源出现在 UI 信源列表。

异常输入：

```text
帮我添加 http://localhost:8080/rss 到融媒智策。
```

预期结果：

- 工具返回失败或 Agent 说明该 URL 被安全校验拒绝。
- 不应绕过校验。

### 4.4 手动抓取

输入：

```text
帮我拉取融媒智策台海观察套餐的最新新闻。
```

预期结果：

- Agent 调用 `media_strategy_ingest`，参数类似 `{"package_ids":["taiwan"]}`。
- 回复包含抓取源数量、新增条数、失败源数量。

### 4.5 热点雷达

输入：

```text
帮我列出融媒智策过去 24 小时台海观察的热点雷达，给 20 条，保留原文链接。
```

预期结果：

- Agent 调用 `media_strategy_hot_radar`。
- 回复标题、source_id、score/risk、原文链接。
- 对高风险内容提醒“仅作线索，需复核”。

### 4.6 Top 选题推荐

输入：

```text
帮我推荐今天最值得做的 5 条融媒选题，只看台海观察，至少 2 家媒体同时报道的优先。
```

预期结果：

- Agent 调用 `media_strategy_top_topics`。
- 参数类似：`package_id=taiwan`、`since_hours=24`、`limit=5`、`min_coverage=2`、`compact=true`。
- 回复只需要标题、原文链接、报道家数、风险等级、article_id，不要把每条新闻扩写成长摘要。

如果返回为空：

- Agent 应先调用 `media_strategy_ingest`，再重试 Top 选题。
- 或建议把 `min_coverage` 降到 1、把时间窗口扩大到 72 小时。

### 4.7 搜索新闻

## 5. API 辅助自测

API 是前端和插件后端通信的接口。普通用户主要通过 UI 测试即可；如果需要定位问题，可以用浏览器开发者工具或命令行访问插件 API。

说明：插件 UI 使用 `window.OpenAkitaPlugin.apiBase` 自动确定基础地址，不同宿主环境下完整 URL 可能不同。下面只列相对路径。

| 功能 | 方法与路径 | 观察点 |
|---|---|---|
| 健康检查 | `GET /healthz` | `ok=true`、版本、源数量、依赖状态 |
| 设置 | `GET /settings` | 当前设置和数据目录 |
| 保存设置 | `PUT /settings` | `reload_required` 是否符合预期 |
| 存储统计 | `GET /storage/stats` | 大小、文件数 |
| 套餐列表 | `GET /packages` | 内置/自定义套餐 |
| 订阅套餐 | `POST /packages/subscribe` | 启停状态 |
| 信源列表 | `GET /sources` | enabled、authority、last_status |
| 添加信源 | `POST /feeds` | URL 安全校验 |
| 抓取 | `POST /ingest` | inserted、failed、errors |
| 热点雷达 | `GET /radar` | items、stats |
| AI 选题分析 | `POST /ai/analyze-top` | 后台 task、progress、pipeline_step |
| 任务列表 | `GET /tasks` | done/failed 状态 |
| 报告列表 | `GET /reports` | 报告历史 |

## 6. 异常与边界用例

### 6.1 网络不可用

操作：

1. 临时断开网络或配置一个不可达代理。
2. 点击 `拉取已启用源`。

预期结果：

- 任务不应卡死。
- 失败源数量增加。
- 信源列表 `last_status` 显示 failed。
- 错误中包含 `network`、超时或连接失败信息。

### 6.2 Brain 不可用

操作：

1. 在大模型未配置或不可用时生成简报。
2. 再生成复核和策研采编计划。

预期结果：

- 不应崩溃。
- 报告仍生成，`source` 应为 `fallback`。
- 文本说明是规则整理结果，并保留原文链接。

### 6.3 空结果

操作：

1. 雷达页关键词输入一个极少出现的词：`zzzz_openakita_media_strategy_empty_test`。
2. 点击生成热点雷达。

预期结果：

- 页面显示“暂无热点”或空列表。
- 指标热点数为 0。
- 不应报错。

### 6.4 高风险内容处理

操作：

1. 启用 `平台热点` 套餐。
2. 拉取并生成热点雷达。
3. 找到 `risk=high` 或低权重热榜源内容。

预期结果：

- 雷达显示风险等级。
- 复核或 Agent 回复中应提醒“仅作线索，需复核”。
- 不应把热榜内容写成事实定论。

### 6.5 权重边界

操作：

1. 编辑一个自定义信源。
2. 权重拖到 `0.10` 保存。
3. 再拖到 `1.00` 保存。

预期结果：

- 保存成功。
- 信源列表显示对应权重。
- 权重只影响排序和主流/边缘标签，不会过滤文章。

## 7. 完整通过标准

本插件自测通过，需要同时满足：

1. 健康检查正常，插件 UI 可打开。
2. 内置套餐可启停，自定义套餐可新建、编辑、克隆、删除。
3. 内置信源可同步、启停、批量启停；自定义 RSS 可添加、编辑、删除。
4. 非法 URL 被拒绝，不能添加 localhost、私有 IP、非 http/https 地址。
5. 至少一次抓取成功，热点雷达能展示文章卡片和原文链接。
6. 关键词搜索、套餐过滤、时间窗口过滤均可用。
7. 信源复核能生成报告，并保存到报告历史。
8. 早报、午报、晚报、专题简报至少各生成一次。
9. 短视频、采访、深度文章、直播脚本四类策研采编计划至少各生成一次。
10. 报告历史可刷新、可预览，报告 Markdown 文件能在输出目录找到。
11. 设置可保存、重新加载后仍保留；自定义数据目录能提示是否需要重新加载。
12. 聊天中 11 个 `media_strategy_*` 工具均能触发或得到合理错误。
13. 无网络、空结果、Brain 不可用等异常不会造成页面崩溃。

## 9. 用户验收建议

建议至少做两轮：

1. 第一轮按默认数据目录和默认套餐测试，确认基础链路可用。
2. 第二轮启用自定义套餐、自定义信源、自定义数据目录，确认个性化配置可持久化。

测试时不要只看“有没有输出”，还要重点看三件事：

1. 每条新闻是否保留原文链接，方便人工复核。
2. 对台海、政策、金融、冲突类内容是否避免未经证实的定论。
3. 报告是否真正保存到历史和本地 Markdown 文件，而不是只显示在页面上。
