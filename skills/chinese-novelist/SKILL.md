# 中文小说写作助手

Designed for中文小说创作设计的技能，Provides角色塑造、情节架构、对话编写、场景描写和长篇Consistency Management等全方位指导，Supports玄幻、都市、悬疑、言情等多种类型。

## When to Use

- From scratchBuild novel outline和世界观
- Create立体的Character profile
- Write natural生动的人物对话
- Build conforming类型规范的情节结构
- Write immersive感强的场景描写
- Maintain long-form小说的一致性（人物、Timeline、设定）
- Specific genre小说的创作指导
- Break writing瓶颈（卡文时的策略）

## Character Creation Guide

### 1. Character Profile Template

每个重要角色应建立Full档案：

```markdown
# Character profile：[角色名]

## 基础信息
- 姓名/别称：
- 年龄：
- 性别：
- 外貌特征：（3-5 个标志性特征，避免面面俱到）
- 身份/职业：
- 社会地位：

## 性格内核
- Core personality：（用 2-3 个词概括，如「骄傲而脆弱」）
- Public persona：（别人眼中的他/她）
- True self：（真正的自己）
- Character flaw：（致命缺陷，推动冲突）
- Character growth方向：（从 A 到 B 的变化弧线）

## Background story
- Formative childhood事件：（塑造性格的根源）
- 创伤/秘密：（驱动行为的暗线）
- Important relationships：（与其他角色的联系）

## Motivations & Goals
- Stated goal：（角色自己认为想要的）
- True need：（角色真正需要的）
- Core fear：（最害怕失去/面对的事物）

## Behavior patterns
- Catchphrase/语言特征：
- Habitual gesture：（紧张时转笔、思考时摸下巴等）
- Decision tendency：（冲动/谨慎/随性）
- Stress response的方式：
```

### 2. Character Building Principles

**Three Elements of Depth：**

| 层次 | Description | Example |
|------|------|------|
| External behavior | 可观察的言行举止 | 总Yes微笑，对所有人彬彬有礼 |
| Internal motivation | 驱动行为的原因 | 害怕被抛弃，用讨好换取安全感 |
| 潜意识 | 角色自己都未意识到的 | 童年被父母忽视的创伤 |

**Character Traps to Avoid：**
- **Overpowered protagonist**：没有弱点的角色缺乏吸引力。给主角明确的短板
- **Stereotypical sidekick**：每个配角都需要独立的动机，而非仅服务于主角
- **Personality consistency矛盾**：变化需要事件推动，而非随意转变
- **Homogenized dialogue**：不同角色说话方式应有区别（用词、句式、语气）

### 3. Character Dialogue Writing

**Basic principles：**
1. 对话要推动情节或揭示性格，不能只Yes「闲聊」
2. 每个角色有独特的说话方式
3. 潜台词比明说更有力量
4. 适度Use「说」的替代词，但不要过度

**Dialogue Design Dimensions：**

| 维度 | Description | Example |
|------|------|------|
| Vocabulary level | 教育程度和性格的体现 | 学者用成语，街头少年用俚语 |
| Sentence length | 反映情绪和节奏 | 紧张时短句，放松时长句 |
| Information density | YesNo直接表达意图 | 内敛的人说话留余地 |
| Modal particles | 地域和年龄特征 | 「嘛」「呐」「哎」各有不同 |
| Pause/Omission | 犹豫、隐瞒的表现 | `"我其实……算了，没什么。"` |

**潜台词示例：**

差 ❌（太直白）：
```
"我很生气，因为你没有告诉我这件事。"
```

好 ✅（潜台词）：
```
她把杯子轻轻放在桌上，声音平稳得可怕："你还有别的事瞒着我吗？"
```

**Inserting action into dialogue：**
```
"这不公平。"林然转过身，指尖在窗台上划了一下，
像在擦去看不见的灰尘，"你比谁都清楚。"
```

## Plot Architecture: Beginning-Development-Turn-Conclusion

### Four-Act Structure

| 阶段 | 比例 | 功能 | 核心任务 |
|------|------|------|---------|
| **起** | 15-20% | 开端 | 建立世界观，介绍主角，埋设悬念 |
| **承** | 30-35% | 发展 | 展开冲突，角色成长，不断升级挑战 |
| **转** | 30-35% | 高潮 | 核心冲突爆发，Maximum危机，反转 |
| **合** | 15-20% | 结局 | 解决冲突，收束伏笔，角色蜕变 |

### Beginning: Three Opening Methods

1. **Event opening**：从一个意外/冲突开始
   ```
   陈然推开门的那一刻，整个办公室安静了。
   所有人都在看着他桌上那封印着红章的信。
   ```

2. **Suspense opening**：抛出谜题，制造好奇
   ```
   三年前的那个雨夜，有三个人走进了那栋楼。
   只有两个人走出来。第三个人，至今没有找到。
   ```

3. **Slice-of-life opening**：从平凡生活切入，反衬后续变化
   ```
   叶青每天早上七点起床，坐三站地铁，
   在便利店买一杯美式。这个习惯保持了两年，
   直到那天早上，地铁里出现了那个人。
   ```

### Development: Pacing Control

**Tension-Relief Principle：**
```
紧张事件 → 缓冲（日常/回忆/感情线）→ 更紧张的事件 → 缓冲 → 高潮
```

**Methods to escalate conflict：**
- Time pressure（deadline 逼近）
- Interpersonal pressure（信任破裂、关系变化）
- Information pressure（新线索改变局面）
- Capability pressure（现有能力不足以应对）
- Moral pressure（两难抉择）

### Turn: Building Climax

高潮不只Yes「大战」，核心Yes**角色面对Maximum考验时的选择**。

**高潮公式：**
```
角色的核心弱点 × Maximum的外部威胁 = 被迫改变/突破
```

**Reversal design techniques：**
- Foreshadowing payoff：前文看似无意的细节在此揭示真相
- Perspective reversal：Switch视角后发现真相与想象不同
- Dilemma choice：没有完美选项，选择即牺牲
- Identity reveal：关键人物真实身份曝光

### Conclusion: Handling Endings

| 结局类型 | 适用场景 | 要点 |
|---------|---------|------|
| Happy ending | 言情、轻松题材 | 满足读者期待，但不要太简单 |
| Open ending | 文艺、悬疑题材 | 给读者想象空间 |
| Tragic ending | 严肃文学 | 必须有足够的情感铺垫 |
| Plot twist ending | 悬疑、惊悚 | 最后的翻转需有充分伏笔支撑 |

## Scene Description Techniques

### Five Senses Description

每个重要场景至少调动 **3 种以上感官**：

| 感官 | 功能 | Example |
|------|------|------|
| 视觉 | 空间感、色彩、光影 | `夕阳把窗框的影子拉成细长的黑线` |
| 听觉 | 氛围、暗示 | `雨点砸在铁皮棚顶上，像密集的鼓点` |
| 嗅觉 | 记忆触发、环境真实感 | `空气里弥漫着消毒水和旧木头的气味` |
| 触觉 | 体感、情绪传递 | `他的手指碰到信封时，纸张的粗糙让他一怔` |
| 味觉 | 氛围强化 | `嘴里全Yes铁锈的味道` |

### Scene-Emotion Resonance

环境描写应服务于角色情绪，而非单纯记录场景：

**焦虑场景：**
```
走廊里的日光灯一闪一闪，发出细微的嗡鸣。
墙上的时钟走得异常缓慢，秒针每跳一格，
他都觉得自己的心跳漏了半拍。
```

**释然场景：**
```
风从半开的窗户挤进来，翻动了桌上几张纸。
远处有人在弹吉他，断断续续的旋律模糊又温柔。
她终于松开了一直攥着的手。
```

### Scene Description Pacing

- **Action scene**：短句，少形容，多动词
- **Atmosphere scene**：长句，多感官，慢节奏
- **Transition scene**：简洁交代，一两句带过
- **Emotional scene**：内心独白+外部细节交织

## Novel Consistency Management

### Setting Document

维护一份随时Update的Setting document：

```markdown
# 《小说名》Setting document

## Timeline
| 章节 | 时间 | 主要事件 |
|------|------|---------|
| 第1章 | 2026年3月 | 主角发现异常 |
| 第2章 | 2026年3月 | 调查开始 |
...

## Character relationship map
主角 ←好友→ 角色A
主角 ←暗恋→ 角色B
角色A ←竞争→ 角色C

## World building
- 地理：
- 组织架构：
- 力量体系：
- 历史背景：

## Foreshadowing tracker
| 伏笔 | 埋设章节 | 回收章节 | 状态 |
|------|---------|---------|------|
| 主角左手的疤 | 第3章 | 第28章 | 已回收 |
| 黑衣人的真实身份 | 第5章 | - | 待回收 |

## 已Use的名称
[避免重复Use类似名字]
```

### Consistency Checklist

- 角色性格YesNo前后一致（变化需要事件推动）
- TimelineYesNo存在矛盾
- 地理距离和旅行时间YesNo合理
- 伏笔YesNoAll回收
- 次要角色YesNo中途消失
- 力量/能力体系YesNo有逻辑漏洞

### Genre Writing Guide

### Fantasy Fiction

**Core elements：**
- Full的力量体系（Clear tier system，Advancement有规则）
- Expansive worldview（多势力、多地域）
- Protagonist growth arc（升级节奏和关键战斗）

**力量体系设计原则：**
1. 等级不超过 7-9 层（太多容易混乱）
2. 每个等级有质变而非量变
3. 跨级战斗需要合理解释（特殊天赋/秘术/代价）
4. 战斗描写避免纯数值对比，融入策略和创意

**节奏控制：**
```
小危机（3-5章）→ 突破/升级 → 中危机（10-15章）→ 
重要突破 → 大危机（20-30章）→ 重大蜕变
```

**常见问题：**
- 升级过快导致后期乏力 → 设计平台期和瓶颈
- 配角沦为NPC → 给重要配角独立的故事线
- 战斗描写同质化 → 不同境界不同战斗方式

### Urban Fiction

**Core elements：**
- Authentic and believable的都市生活细节
- 具体的Career description（行业知识）
- Relationship network的复杂性
- Social issues的融入

**细节真实感：**
```
地铁在早高峰时的拥挤程度、
特定城市的街道名和地标、
某个行业的术语和工作流程、
不同社会阶层的消费习惯
```

**人物关系处理：**
- 职场关系：上下级、同事竞争、导师
- 家庭关系：代际冲突、亲情羁绊
- 感情关系：自然发展，避免强行牵线
- 社交圈层：不同圈层的价值观碰撞

### Suspense Fiction

**Core elements：**
- Tight logical chain
- Effective misdirection和反转
- 线索的Fair presentation
- 节奏控制（Timing of information release）

**线索设计三原则：**
1. **公平原则**：关键线索必须在揭示真相前呈现给读者
2. **迷惑原则**：真线索和假线索混合，引导读者误判
3. **连贯原则**：最终答案必须能串联所有线索

**悬念维持技巧：**
- 每章结尾留一个小钩子
- 解答一个疑问的同时抛出新疑问
- 利用视角限制制造信息差
- 关键时刻Switch视角/Timeline

**反转层次：**
```
第一层反转：推翻读者最初的猜测（第1/3处）
第二层反转：推翻读者修正后的猜测（第2/3处）
第三层反转：真相揭示，意料之外情理之中（结尾）
```

### Romance Fiction

**Core elements：**
- 让人心动的化学反应
- Reasonable emotional development pace
- Memorable interaction scenes
- Character growth interwoven with romance

**Romance development pace：**
```
相遇（吸引/排斥）→ 接触（了解/误解）→ 亲近（心动/试探）→
阻碍（误会/外部压力）→ 确认（表白/相互理解）→ 磨合（冲突/成长）→
稳定（彼此接纳真实的对方）
```

**Heart-flutter scene design tips：**
- Via细节传递心动（不Yes直接写「她心跳加速」）
- 利用日常场景营造氛围（一起做饭比烛光晚餐更动人）
- 未完成的动作比完成的更撩人
- 眼神交流和无意触碰比大段心理独白更有效

**Classic romantic scene patterns：**

| 模式 | Description | Example |
|------|------|------|
| Synchronized moment | 异口同声/心有灵犀 | 同时说出一样的话然后对视 |
| Contrast charm | 平时形象vs特定场景 | 冷面上司认真给猫喂食 |
| Hero saves beauty/美救英雄 | 危机时刻的保护 | 不一定Yes物理危险 |
| Jealousy/Misunderstanding | 情绪波动揭示感情 | 说「没关系」但表情出卖了自己 |
| Accidental intimacy | 物理距离突然缩短 | 避雨时共撑一把伞 |

## Breaking Writer's Block

| 困境 | 策略 |
|------|------|
| 不知道下一步写什么 | 回到角色动机，问「TA 下一步会做什么」 |
| 场景写不下去 | 换视角写同一场景 |
| 对话写不自然 | 把对话念出声，删掉不会说出口的 |
| 感觉情节太平淡 | 加入一个意外事件或提前引爆冲突 |
| 角色扁平 | 给角色加一个秘密或未说出的恐惧 |
| 世界观太空 | 从一个具体地点的细节开始构建 |
| 结尾写不好 | 回看开头，让结尾与开头形成呼应 |
